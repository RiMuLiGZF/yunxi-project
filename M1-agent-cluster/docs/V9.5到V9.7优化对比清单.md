# V9.5→V9.7 优化对比清单

> 版本对比：V9.5 vs V9.7
> 迭代轮次：2轮增量优化（V9.6 第一轮5项修复 + V9.7 第二轮2项修复）
> 测试覆盖：541项测试全量通过
> 日期：2026-06-28

## 一、修复总览表

| 维度 | V9.5 状态 | V9.7 状态 | 提升 |
|------|----------|----------|------|
| Ledger数据闭环 | `close_task()`未实现，task_ledger永久驻留内存 | `close_task()`自动归档：未完成任务→FINAL、移除active集合、保留ProgressLedger只读存档、清理replan_counts | 内存安全+数据闭环 |
| Registry能力查询 | O(N)遍历所有注册Agent | `_capability_index`反向索引，capability→set(agent_id)，O(1)查找 | 查询性能10-100x |
| 重试退避策略 | 固定指数退避（base_delay * 2^retry_count） | 按错误类型五类独立配置 + EMA动态学习乘数 + 混合延迟计算 | 智能化自适应 |
| V8 tracer获取 | `__getattr__`深层穿透6层链式获取tracer | `__init__`显式提取存储到`self._tracer`，process()使用本地引用 | 解耦+稳定性 |
| A2A Task安全 | 仅AgentCard签名（发现阶段），Task无签名 | Task级HMAC-SHA256签名 + HTTPTransport自动签名/验签 | 端到端安全 |
| 重试EMA学习 | 无 | 连续失败增大乘数（上限4.0）、成功后减小乘数（下限1.1）、alpha=0.3 | 自我优化 |
| HTTPTransport签名 | 无 | `send()`自动签名，`subscribe()`响应自动验签 | 传输层安全 |
| Ledger TTL清理 | 仅清理replan_counts | 新增孤立progress ledger清理（task已关闭且无引用） | 更彻底内存治理 |

## 四、性能影响对比

| 操作 | V9.5 | V9.7 | 改善 |
|------|------|------|------|
| Registry.find_by_capability | O(N)遍历所有Agent | O(1)索引查找 | ~100x @ 1000 Agent |
| Ledger内存占用 | task_ledger永久驻留，无上限 | close_task自动移除，TTL清理孤立ledger | 避免长期运行内存泄漏 |
| Retry延迟计算 | 固定 `base * 2^retry_count` | `(profile_multiplier + learned_multiplier) / 2`混合计算 | 更贴合实际错误模式 |
| V8.process() tracer访问 | 6层链式穿透 getattr | 本地属性访问 | 减少5次属性解析开销 |
| HTTPTransport.send() | 无签名开销 | HMAC-SHA256签名+验签（约0.1ms/次） | 安全增强，开销可忽略 |
| 重试退避准确性 | 固定策略可能过度/不足退避 | EMA学习使策略随时间自我优化 | 长期运行收敛到最优退避 |

**EMA学习收敛示例**：
```
场景：某Agent频繁网络超时
V9.5固定策略：每次重试延迟 = 1.0 * 2^n → 1s, 2s, 4s, 8s, 16s...
V9.7 EMA学习：初始learned_multiplier=2.0，连续失败后逐渐增大至3.0+
  → 混合乘数 = (2.0 + 3.0) / 2 = 2.5
  → 延迟 = 1.0 * 2.5^n → 1s, 2.5s, 6.25s, 15.6s...
  → 更激进退避，减少无效重试

场景：系统恢复稳定后
V9.5固定策略：仍使用2^n，退避过度
V9.7 EMA学习：成功后learned_multiplier从3.0衰减至1.5
  → 混合乘数 = (2.0 + 1.5) / 2 = 1.75
  → 延迟 = 1.0 * 1.75^n → 1s, 1.75s, 3.06s, 5.36s...
  → 更积极重试，降低恢复延迟
```

## 五、专利创新点演进

### V9.5 专利创新点（3项）

| 编号 | 创新点 | 技术方案 | 局限 |
|------|--------|---------|------|
| 1 | TLS-Vis三层分级可见性 | guest/member/owner动态过滤 + 动作词标记 | 稳定，无变化 |
| 2 | Safe-Ledger双层防死循环 | plan级5次+任务级20次+SKIPPED终态 | close_task未实现，内存泄漏风险 |
| 3 | Heuristic-Stall-Detector | 零进展/循环依赖/孤儿计划纯规则检测 | 稳定，无变化 |

### V9.7 专利创新点（5项，+2项新增）

| 编号 | 创新点 | 演进路径 | V9.7强化内容 |
|------|--------|---------|-------------|
| 1 | Safe-Ledger双层防死循环 | V9.5基础 → V9.6 close_task闭环 → V9.7稳定 | `close_task()`实现完整生命周期，TTL清理孤立ledger，数据闭环 |
| 2 | TLS-Vis三层分级可见性 | V9.5稳定 → V9.7无变化 | 维持V9.5设计 |
| 3 | Heuristic-Stall-Detector | V9.5稳定 → V9.7无变化 | 维持V9.5设计 |
| 4 | **Adaptive-Retry自适应重试** | V9.5基础RetryCoordinator → V9.6错误类型分类 → **V9.7 EMA学习** | 新增EMA动态学习退避乘数，比固定指数退避更智能 |
| 5 | **A2A-Task-Signed端到端签名** | V9.5仅AgentCard签名 → V9.6 Task签名API → **V9.7 HTTPTransport自动集成** | 弥补A2A v1.0仅AgentCard签名的安全缺口，实现Task级端到端保护 |

**创新点4演进详解**：
```
V9.5: RetryCoordinator统一协调CB/DLQ/Ledger，但使用固定base_delay * 2^retry_count
V9.6: 引入五类错误独立配置（timeout/network/oom/circuit_breaker/unknown），实例级max_delay上限
V9.7: 引入learned_multiplier EMA学习：
  - record_retry(): retry_count>=2时，learned_multiplier *= (1 + 0.3*0.3)，上限4.0
  - record_success(): learned_multiplier *= (1 - 0.3*0.5)，下限1.1
  - delay = base * ((profile_multiplier + learned_multiplier)/2)^(retry_count-1)
```

**创新点5演进详解**：
```
V9.5: A2A Protocol适配层完整，但安全仅到AgentCard级别
V9.6: Task模型新增signature字段，a2a_protocol.py提供sign_task()/verify_task()
V9.7: HTTPTransport集成签名：
  - send(): 若配置了secret且task无signature，自动调用sign_task()
  - 接收响应：若包含signature且配置了secret，自动调用verify_task()
  - 验签失败：返回TaskUpdate(status=FAILED, error="Task signature verification failed")
```
