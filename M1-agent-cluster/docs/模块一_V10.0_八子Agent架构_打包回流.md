## 一、模块完整架构说明

### 1.1 核心定位

模块一（M1）是云汐系统的**多Agent集群调度中枢**。V10.0将单一编排器重构为**8个专属子Agent**架构，每个子Agent承担独立职责，通过A2A协议协作，实现14项核心功能。

### 1.3 八子Agent职责对照表

| # | 子Agent | 文件目录 | 核心职责 | 行业吸收 |
|---|---------|----------|---------|---------|
| 1 | Orchestrator-Agent | `orchestrator/` | DAG构建、任务解析、拓扑排序、Agent能力匹配 | Dify可视化编排 |
| 2 | Lifecycle-Agent | `lifecycle/` | Agent全生命周期管理、引用计数、优雅终止 | LangGraph显式状态机 |
| 3 | Discovery-Agent | `discovery/` | 注册发现、反向索引、负载评分、端云调度决策 | K8s调度评分模型 |
| 4 | Bus-Agent | `bus/` | A2A消息路由、优先级队列、死信队列管理 | Kafka分区路由 |
| 5 | Snapshot-Agent | `snapshot/` | 增量快照、快照链、断点续跑、SHA256完整性校验 | K8s RAG增量快照 |
| 6 | Budget-Agent | `budget/` | 滚动窗口预算(15min/1h/24h)、熔断、成本预警 | — |
| 7 | Security-Agent | `security/` | Prompt注入检测、PII脱敏、涉密四级分级、审计留痕 | CrewAI角色隔离 |
| 8 | Arbiter-Agent | `arbiter/` | Wait-For图死锁检测、三级冲突仲裁、人工介入 | AutoGen多Agent仲裁 |

## 四、新旧版本对比

| 维度 | V9.9（旧架构） | V10.0（新架构） | 提升 |
|------|---------------|----------------|------|
| 架构模式 | 七层委托链（V2→V9） | 8子Agent + A2A协议 | 职责清晰度大幅提升 |
| 任务编排 | OrchestratorV9.process()单一入口 | DAGBuilder → TaskDAG拓扑排序 | 支持多Agent并行协作 |
| Agent生命周期 | 简单register/unregister | CREATED→ARCHIVED八态状态机+引用计数 | 优雅终止+资源回收 |
| 负载均衡 | 简单轮询 | VRAM/CPU/电量/网络四维综合评分 | 智能调度决策 |
| 端云协同 | 无 | LOCAL_FIRST/AUTO/CLOUD_FIRST决策树 | 离线缓存+重连同步 |
| 涉密安全 | GuardrailsV2安检+PII脱敏 | 四级分级+权限域隔离+审计留痕 | 满足等保要求 |
| 死锁防护 | CancelToken任务取消 | Wait-For图+三级仲裁+人工介入 | 系统级死锁防护 |
| 快照恢复 | 无 | 增量快照链+SHA256校验+断点续跑 | 任一时刻可回溯 |
| 分身管理 | 无 | 勘探/规划/撰写/审查四种分身+最小信息 | 安全高效的临时委派 |
| 代码组织 | 40+文件扁平目录 | 9个子模块独立目录 | 可维护性提升 |
