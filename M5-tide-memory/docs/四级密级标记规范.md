# 四级密级标记规范

> ⚠️ 本规范为M5潮汐记忆系统内部安全规范

## 密级定义

| 密级 | 标识 | 存储方式 | 同步策略 | 访问角色 |
|------|------|----------|----------|----------|
| 公开级 | PUBLIC | 明文 | 可同步云端 | 所有人 |
| 内部级 | INTERNAL | 明文 | 可同步云端 | 内部成员 |
| 机密级 | CONFIDENTIAL | AES加密 | 加密同步 | 授权人员 |
| 绝密级 | TOP_SECRET | AES-256-GCM | **不同步** | 最高权限 |

## 标记规则

### 自动标记

1. **默认密级**：所有新记忆默认标记为 `TOP_SECRET`
2. **降级规则**：根据保留时间自动降级
   - TOP_SECRET → CONFIDENTIAL：365天
   - CONFIDENTIAL → INTERNAL：180天
   - INTERNAL → PUBLIC：90天
3. **升级规则**：满足以下条件可手动升级
   - 涉及用户隐私信息
   - 高情绪强度记忆
   - 系统核心配置

### 手动标记

API支持显式指定密级：
```python
from tide_memory.security.secret_marker import SecretMarker, ClassificationLevel

sm = SecretMarker()
sm.mark("memory_id", ClassificationLevel.CONFIDENTIAL, reason="包含用户隐私")
```

## 访问控制

### 角色定义

| 角色 | 说明 | 可访问密级 |
|------|------|-----------|
| admin | 系统管理员 | 全部 |
| top_secret | 绝密级 | TOP_SECRET及以下 |
| confidential | 机密级 | CONFIDENTIAL及以下 |
| internal | 内部级 | INTERNAL及以下 |
| public | 公开级 | 仅PUBLIC |

### 访问检查流程

```
请求 → 身份认证 → 域权限检查 → 密级检查 → 审计记录 → 返回结果
```

## 加密要求

- **算法**：AES-256-GCM
- **密钥长度**：256位
- **IV长度**：96位（每次加密随机生成）
- **认证标签**：128位
- **密钥派生**：PBKDF2-HMAC-SHA256，100,000次迭代

## 安全删除

- TOP_SECRET级数据删除时执行**安全删除**（3次覆写）
- 覆写模式：0x00 → 0xFF → 0x00
- 删除后不可恢复

## 审计要求

- 所有密级变更必须记录审计日志
- 所有TOP_SECRET访问必须记录审计日志
- 审计日志保留期：不少于180天
