# 变更日志

本项目的所有重要变更均记录在此文件中。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，
版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [0.5.2] - P2 可维护性优化阶段

### 新增
- 数据库连接统一管理模块，所有层级共享同一连接池，消除重复的连接初始化逻辑
- 错误码枚举统一至 `tide_memory.errors.ErrorCode`，M8ErrorCode 作为向后兼容别名
- 潮汐相位管理 API 端点（`/api/v1/memory/phase`、`/api/v1/memory/phase/switch`）
- 类型注解补齐，核心模块函数签名全覆盖
- 测试用例补齐：数据库迁移测试、全量集成测试、增长子系统测试、中间件测试、技能接口测试、向量检索 GPU 测试

### 变更
- 重构 M8 接口适配层，错误码唯一来源改为 `ErrorCode` 枚举
- 统一配置加载路径，环境变量与 YAML 配置合并策略标准化
- 模块导入规范化，消除循环依赖风险

### 修复
- 修复并发场景下数据库连接泄漏问题
- 修复配置缺失时启动崩溃的边界情况

## [0.5.1] - P1 健壮性提升阶段

### 新增
- 熔断器中间件（`circuit_breaker`），对外部依赖实现自动熔断与恢复
- 限流中间件（`rate_limit`），支持令牌桶与滑动窗口算法
- 重试机制模块（`retry`），支持指数退避与可配置重试策略
- 事务管理器（`transaction`），为记忆写操作提供原子性保证
- 向量删除优化，批量删除时同步更新索引，避免无效条目残留
- 输入验证层，所有 API 入参经过 Pydantic Schema 校验

### 变更
- 魔法常量治理：硬编码数值统一迁移至 `constants.py` 集中管理
- 长函数拆分：`recall_engine`、`consolidation`、`l3_abyss` 等模块的单函数行数控制在合理范围
- 认证中间件增强，支持多级 Token 校验与过期策略

### 修复
- 修复高并发下幂等键竞争导致的重复写入
- 修复向量索引在异常中断后一致性损坏的问题

## [0.5.0] - P0 基础夯实阶段

### 新增
- 全局异常处理中间件（`exception_handler`），统一捕获未预期异常并返回标准错误响应
- 日志统一模块（`logging_setup`），基于 structlog 实现结构化日志输出
- 幂等性中间件（`idempotency`），防止网络重试导致的重复操作
- JWT 认证中间件（`auth`），Bearer Token 鉴权机制
- 数据库迁移框架（`migration`），支持版本化 schema 演进
- 配置校验模块（`config_schema`），启动时验证必要配置项完整性
- 审计日志模块（`audit_logger`），记录所有敏感操作的完整审计链

### 新增（安全）
- 数据脱敏模块（`desensitizer`），对外返回内容自动脱敏
- 四级密级标记系统（PUBLIC / INTERNAL / CONFIDENTIAL / TOP_SECRET）
- 三级域权限隔离（private / shared / core）
- AES-256-GCM 加密模块，TOP_SECRET 级别数据强制加密存储

### 新增（记忆核心）
- 四层潮汐记忆模型架构（L0 沙滩 / L1 浅水 / L2 深水 / L3 深海）
- 情绪推断引擎（EI），基于效价-唤醒度模型
- 睡眠巩固调度器，支持多模式记忆迁移
- 缓存协调器（`cache_coordinator`），层级间缓存一致性管理
- 关键词检索与向量检索双引擎
- M8 标准接口适配层

### 新增（增长子系统）
- 成长日历、成就系统、天赋体系
- 编年史与回声模块
