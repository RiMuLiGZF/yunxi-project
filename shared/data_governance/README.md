# 数据治理模块

云汐系统数据治理规范与工具集。

## 模块结构

```
data_governance/
├── __init__.py                      # 模块入口
├── sovereignty.py                   # 数据主权查询工具
├── data_sovereignty.json            # 数据主权清单（权威配置）
└── deduplication_plans/             # 去重执行计划
    ├── index.json                   # 去重计划总索引
    └── P0_watch_migration.json      # P0 手表域迁移计划
```

## 数据主权三原则

1. **谁产生、谁拥有、谁维护**：每个模块对自己产生的数据拥有完全主权
2. **禁止跨库直连**：跨模块数据交互必须通过 HTTP API 或消息队列
3. **权威数据源唯一**：每个数据域只有一个权威拥有者，其他模块均为消费者

## 使用方式

### 查询数据域权威拥有者

```python
from data_governance import check_data_owner

owner = check_data_owner("life")  # 返回 "M4"
owner = check_data_owner("growth")  # 返回 "M5"
owner = check_data_owner("watch")  # 返回 "M6"
```

### 获取模块主权信息

```python
from data_governance import get_module_sovereignty

m8_info = get_module_sovereignty("M8")
print(m8_info["core_domains"])  # M8 核心数据域
print(m8_info["tables_to_deprecate"])  # 应废弃的业务表
```

### 获取去重任务清单

```python
from data_governance import list_overlapping_domains

tasks = list_overlapping_domains()
for task in tasks:
    print(f"[{task['priority']}] {task['domain']}: {task['table_count']}张表 -> {task['target_module']}")
```

## 去重优先级

### P0 - 立即执行（低风险练手）
- **手表域 (4张表) → M6**：边界最清晰，M6 本来就是硬件外设模块

### P1 - 高优先级（核心业务数据）
- **生活管理 (9张表) → M4**：M4 已有完整对应表
- **学业规划 (7张表) → M4**：M4 已有完整对应表
- **工作开发 (4张表) → M9**：M9 需要扩充模型

### P2 - 中低优先级（逐步推进）
- 复盘总结 (5张表) → M4
- 人际关系 (4张表) → M4
- 情绪陪伴 (8张表) → M4/M5
- 形象工坊 (5张表) → M4
- 成长系统 (8张表) → M5（底层存储整合）
- 工作流引擎 (2张表) → M7

## 迁移方法论

所有数据迁移统一采用 **四步迁移法**：

```
Step 1: 准备阶段
  ├─ 目标模块建表 + API
  ├─ 编写迁移脚本 + 对账脚本
  └─ 接入统一备份

Step 2: 双写阶段
  ├─ 写操作同时写入两边
  ├─ 历史数据全量迁移
  └─ 每日对账 + 不一致修复

Step 3: 切读阶段
  ├─ 非关键路径先切
  ├─ 核心路径逐步切换
  └─ 保留缓存（可选）

Step 4: 停写清理
  ├─ 停止源端写操作
  ├─ 7天观察期
  ├─ 备份旧表
  └─ 删除 + 更新文档
```

## 配置文件说明

### data_sovereignty.json
数据主权清单的权威配置。定义了每个模块的：
- 核心数据域
- 拥有的表
- 与其他模块的重叠关系
- 待废弃的表

**注意**：此文件是权威配置，修改需谨慎。修改后应同步更新相关模块代码。

### deduplication_plans/index.json
去重执行计划总索引。按优先级排列所有去重任务，包含：
- 每个批次的表数量、目标模块、难度、预估工期
- 通用迁移步骤模板
- 迁移原则

### P0_watch_migration.json
P0 批次（手表域）的详细迁移计划，包含：
- 目标表结构设计
- 字段映射关系
- API 设计
- 风险评估与回滚方案
- 成功标准

## 维护与更新

- 新增模块时，在 `data_sovereignty.json` 中登记其数据主权
- 新增业务表时，确认其所属模块，避免跨模块重复建表
- 去重任务完成后，更新对应计划文件的 status 字段
- 每季度审查一次数据主权清单，确保与实际架构一致
