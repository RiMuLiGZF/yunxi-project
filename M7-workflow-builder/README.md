# M7 积木编排平台 (Workflow Builder)

**模块代号**：M7
**模块名称**：积木编排平台
**版本**：v1.0.0
**端口**：8007
**技术栈**：FastAPI + DAG执行引擎 + JSON存储

---

## 一、模块概述

M7 积木编排平台是云汐系统的可视化工作流编排工具，用户可以通过拖拽"积木"的方式构建自动化工作流程，支持条件分支、循环、子工作流等高级特性。平台内置多种基础积木，并可对接 M2 技能集群扩展更多能力。

### 核心能力

| 能力 | 说明 |
|------|------|
| **可视化编排** | 拖拽式工作流编辑器，所见即所得 |
| **DAG 执行** | 基于拓扑排序的有向无环图执行引擎 |
| **多类型积木** | 内置 8+ 基础积木 + M2 技能积木扩展 |
| **模板库** | 5 个内置工作流模板，一键应用 |
| **运行调试** | 分步执行、变量查看、日志追踪 |
| **M8 标准对接** | 完整实现 /m8/health、/m8/metrics、/m8/config |

## 三、积木类型

### 内置积木（8种）

| 积木 | 说明 |
|------|------|
| **开始/结束** | 工作流起止节点 |
| **HTTP 请求** | 发起 HTTP 请求调用外部 API |
| **条件判断** | 基于条件的分支执行 |
| **变量设置** | 设置/修改工作流变量 |
| **代码执行** | 执行 Python/JS 代码片段 |
| **延时等待** | 等待指定时间 |
| **数据转换** | JSON/文本数据转换 |
| **子工作流** | 调用其他工作流 |

### M2 技能积木（需 M2 可用）

对接 M2 技能集群后，可使用 20+ 种技能积木，包括：
- 日历管理、待办事项、目标管理
- 全文搜索、文档处理、数据分析
- 翻译、闪卡、习惯追踪、心情日记
- 财务、联系人、通知、代码搜索

## 六、工作流执行原理

### DAG 执行引擎

1. **构建图**：将工作流的积木和连线解析为有向无环图（DAG）
2. **拓扑排序**：使用 Kahn 算法计算执行顺序
3. **逐步执行**：按拓扑顺序依次执行每个积木
4. **变量传递**：上游积木的输出作为下游积木的输入
5. **错误处理**：单积木失败不影响其他分支，支持重试

### 执行状态

| 状态 | 说明 |
|------|------|
| `pending` | 等待执行 |
| `running` | 正在执行 |
| `completed` | 执行成功 |
| `failed` | 执行失败 |
| `cancelled` | 已取消 |

## 七、与其他模块关系

```
┌─────────────┐  技能扩展    ┌─────────────┐
│  M2 技能集群 │ ◀────────── │  M7 积木平台 │
└─────────────┘              └──────┬──────┘
                                    │
                              M8 纳管
                                    │
                              ┌─────▼─────┐
                              │ M8 管理台 │
                              └───────────┘
```

- **上游**：M8 管理台通过 M8 标准接口纳管 M7
- **下游**：M7 调用 M2 技能集群扩展积木能力
- **前端**：M7 前端（m7/）通过 API 交互
