# YX-CHG-0003 — M9 修复 uptime_seconds 硬编码问题

- **编号**：YX-CHG-0003
- **日期**：2026-07-12
- **模块**：M9 开发工坊
- **优先级**：P1
- **来源**：第一阶段修复（体检报告发现）

---

## 改动背景

体检报告发现 M9 开发工坊的 M8 标准健康检查接口中，`uptime_seconds` 硬编码为 0，未实际统计运行时间，导致监控数据不准确。

## 改动内容

**文件**：`M9-programming-dev/src/m9_programming_dev/main.py` 第 62 行附近

修改前：
```python
"uptime_seconds": 0,
```

修改后：
```python
import time

_start_time_m8 = time.time()

# ... 在 m8_health 处理函数中 ...
"uptime_seconds": int(time.time() - _start_time_m8),
```

**实现方式**：
- 模块加载时记录启动时间戳 `_start_time_m8`
- 健康检查接口返回时动态计算当前时间与启动时间的差值
- 与 M6、M10、M8 等模块的实现方式保持一致

## 影响范围

- 影响接口：`/m8/health`
- 影响功能：M8 管理台的服务运行时长监控
- 风险等级：低（无逻辑变更，仅修复数据准确性）

## 验证方式

1. 启动 M9 服务
2. 等待几秒后访问 `http://localhost:8009/m8/health`
3. 确认 `uptime_seconds` 值大于 0，且随时间递增

修改量小，通过 git 回滚即可。
