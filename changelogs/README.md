# 云汐系统 — 改动记录仓库

> 本目录记录云汐系统的所有迭代改动，每次改动都有对应的记录文档。
> 配合 Git 使用，Git 记录代码变更，本目录记录改动说明。

---

## 编号规则

```
YX-CHG-{序号}-{模块}-{简要描述}-{日期}.md
```

- **序号**：4 位数字，从 0001 开始递增
- **模块**：涉及的主要模块（如 M1 / M8 / 全系统）
- **日期**：改动日期（YYYYMMDD）

示例：`YX-CHG-0001-M6-修复SSE导入路径错误-20260712.md`

---

## 记录模板

每份改动记录包含以下内容：

1. **基本信息** — 改动编号、日期、涉及模块、优先级
2. **改动背景** — 为什么要改，问题来源（体检报告/bug反馈/需求）
3. **改动内容** — 具体改了什么，改前改后对比
4. **影响范围** — 影响哪些模块/功能
5. **验证方式** — 如何验证改动有效
6. **回滚方案** — 出问题如何回滚
7. **关联文档** — 相关的体检报告、需求文档等

## 使用说明

### 新增改动

1. 完成代码改动
2. 在本目录创建对应编号的改动记录
3. 更新本 README 的改动列表
4. Git 提交代码 + 改动记录

### 查看改动

- 按编号顺序查看，了解迭代脉络
- 按模块筛选，查看特定模块的改动历史
- 配合 `git log` 使用，代码变更 + 改动说明对照
