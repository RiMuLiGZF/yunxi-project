## 一、背景

根据系统体检报告，云汐项目中存在大量历史备份和废弃代码，导致：

- 项目体积膨胀，维护成本高
- 新成员难以分辨哪些是有效代码
- Git 仓库体积过大，克隆和检索效率低
- 容易误用废弃代码导致问题

当前已有完整的 M11 版系统备份（存放在 `../云汐系统备份/云汐系统M11版/`），可以安全清理项目内的冗余备份和废弃目录。

## 四、影响范围

### 4.1 正面影响
- 项目目录更清晰，易于维护
- 减少误用废弃代码的干扰
- 降低 Git 仓库体积
- 提升代码搜索和 IDE 索引速度

### 4.2 风险控制
- 清理前已确认有完整系统备份
- 废弃目录均有 DEPRECATED 标识，确认无引用
- 备份文件均为 .bak 后缀，不参与运行
- 保留 `__pycache__` 和 `.pytest_cache` 可自动重建
