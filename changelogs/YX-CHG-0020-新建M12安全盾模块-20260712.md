## 一、变更背景

### 1.1 问题分析

云汐系统现有安全能力存在以下问题：

1. **碎片化严重**：至少 5 套独立的鉴权/审计/限流实现（M8/M10/M11/shared/M5各一套）
2. **缺少 Web 防护**：无 SQL 注入、XSS、CSRF、命令注入等常见 Web 攻击防护
3. **缺少 IP 管控**：无黑白名单、自动封禁、地理控制等 IP 层防护
4. **缺少统一安全运营中心**：各模块安全数据分散，无法全局态势感知

### 1.2 模块定位

**M12 安全盾（Security Shield）** — 云汐系统的统一安全防护中心：
- **WAF 防护墙**：SQL注入/XSS/CSRF/命令注入/路径遍历检测
- **统一鉴权中心**：API Key + JWT + RBAC 角色权限 + 速率限制
- **IP 访问控制**：黑白名单 + 自动封禁 + CIDR 段支持
- **安全审计中心**：事件记录 + 告警 + 多维统计 + 态势感知
- **安全管理界面**：可视化仪表盘 + 规则管理 + 日志查询

### 1.3 与 M10 系统卫士的区别

| 维度 | M10 系统卫士 | M12 安全盾 |
|------|-------------|-----------|
| **核心关注** | 基础设施稳定性、资源可用性 | 信息安全、访问控制、数据防护 |
| **防护对象** | CPU/内存/磁盘/温度/进程 | 用户数据、API 密钥、权限、网络请求 |
| **主要手段** | 阈值告警、过载限流、进程拉起 | WAF、加密、鉴权、IP封禁、审计 |
| **触发条件** | 资源使用率超过阈值 | 未授权访问、攻击检测、越权操作 |
| **定位** | 运维卫士 | 安全卫士 |

两者互补，共同构成云汐系统的防护体系。

## 三、核心功能

### 3.1 WAF 安全防护墙

#### 检测能力（5大类 25+ 规则）

| 类别 | 规则数 | 说明 |
|------|--------|------|
| SQL 注入 | 7 条 | 经典注入、盲注、报错注入、堆叠查询、UNION注入等 |
| XSS 跨站脚本 | 8 条 | 脚本标签、事件处理、SVG/IMG注入、JS伪协议、编码绕过等 |
| 命令注入 | 5 条 | 分号/管道/反引号/&& 等命令拼接、危险命令检测 |
| 路径遍历 | 4 条 | ../ 遍历、绝对路径、编码绕过等 |
| CSRF | 1 条 | 跨站请求伪造检测 |

#### 技术特性
- 多维度检测：Query 参数 + Body + Header + Cookie
- 正则匹配 + 启发式检测
- 规则可配置：启用/禁用、自定义规则、严重级别调整
- 动作可选：拦截/告警/记录
- 性能优化：规则预编译、快速失败

### 3.2 统一鉴权中心

#### 认证方式
- **API Key**：SHA256 哈希存储，前缀识别，支持过期/禁用/轮换
- **JWT Bearer Token**：Access Token + Refresh Token 双令牌
- **账号密码**：bcrypt 哈希存储（管理后台用）

#### 权限体系
- **五级角色**：OWNER > ADMIN > OPERATOR > VIEWER > USER
- **细粒度权限**：Scope 权限范围（waf:read / audit:write 等）
- **角色层级继承**：高级角色自动包含低级角色权限

#### 速率限制
- 令牌桶算法，默认 60 次/分钟，突发 30 次
- 按 IP / API Key 双维度限流
- 支持自定义限流策略
- 自动过期清理

### 3.3 IP 访问控制

#### 功能特性
- **黑白名单**：精确 IP + CIDR 段支持
- **自动封禁**：WAF 检测到攻击后自动加入黑名单
- **封禁时长**：临时(1h) / 短期(24h) / 长期(7d) / 永久 四级
- **白名单豁免**：白名单 IP 跳过所有 WAF 检测和限流
- **命中统计**：自动记录黑名单 IP 的访问次数和最近时间

### 3.4 安全审计中心

#### 审计内容
- 所有安全事件（WAF 拦截、IP 封禁、鉴权失败等）
- 管理操作（规则变更、密钥管理、配置修改）
- 登录登出、权限变更
- 请求/响应全链路追踪

#### 统计分析
- 按时间维度：小时/日/周趋势
- 按类型维度：攻击类型分布
- 按来源维度：IP 排名、地理位置
- 按级别维度：info/warning/critical/emergency 占比

### 3.5 前端管理界面（6个页面）

| 页面 | 功能 |
|------|------|
| `login.html` | 登录页（账号密码 + API Key 两种方式） |
| `dashboard.html` | 安全仪表盘（安全评分、攻击趋势、事件流、来源分布） |
| `waf.html` | WAF 管理（开关、规则列表、自定义规则、拦截统计） |
| `ip-control.html` | IP 控制（黑白名单、自动封禁、封禁配置、IP检测） |
| `auth.html` | 鉴权管理（API密钥、角色权限、使用统计） |
| `audit.html` | 安全审计（事件列表、筛选、统计、导出） |

## 四、安全最佳实践

本模块遵循以下安全最佳实践：

1. **密码安全**：bcrypt 哈希存储，防时序攻击比较
2. **API Key 安全**：SHA256 哈希存储，仅创建时明文展示一次
3. **JWT 安全**：HS256 签名，过期时间控制，刷新令牌机制
4. **输入验证**：所有输入参数 Pydantic 校验
5. **SQL 注入防护**：SQLAlchemy ORM 参数化查询
6. **XSS 防护**：输出编码 + WAF 检测
7. **速率限制**：防止暴力破解和 DDoS
8. **审计日志**：所有敏感操作留痕，防篡改
9. **最小权限**：默认用户权限最小化，按需授权

## 七、使用方式

### 启动服务
```bash
cd M12-security-shield
pip install -r requirements.txt
python server.py
```

### 访问地址
- API 文档：http://localhost:8012/docs
- 管理界面：http://localhost:8012/frontend/login.html

### 环境变量配置
```env
M12_PORT=8012
M12_ENV=development
M12_JWT_SECRET=your-secret-key
M12_ADMIN_USERNAME=admin
M12_ADMIN_PASSWORD=admin123
M12_WAF_ENABLED=true
M12_RATE_LIMIT_ENABLED=true
```
