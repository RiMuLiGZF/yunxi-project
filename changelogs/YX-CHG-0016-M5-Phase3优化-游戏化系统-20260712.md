## 一、背景

Phase 2 完成了核心能力深化（加密原文存储、HNSW索引、WAL优化、缓存联动、四相控制器）。

Phase 3 聚焦**游戏化系统**，完整实现 v2.4 设计中的 6 大游戏化模块，将 M5 从"纯工具型记忆库"升级为"有成长感的认知操作系统"。

前端 6 个游戏化页面已存在于 `frontend/modes/` 目录下：
- growth-achievements.html - 成就勋章殿堂
- growth-talent-tree.html - 心智天赋树
- growth-chronicle.html - 地球Online编年史
- growth-memory-echo.html - 记忆回响对比
- growth-tide-calendar.html - 潮汐专属历法
- growth-season-journey.html - 赛季征程

本次 Phase 3 实现对应的后端 API，让前端页面能真正跑起来。

### 2.2 成就勋章殿堂（achievements）

**API 端点（3个）**：
- `GET /api/v1/growth/achievements` - 成就列表（支持 category、status 过滤）
- `GET /api/v1/growth/achievements/stats` - 成就统计
- `POST /api/v1/growth/achievements/{id}/unlock` - 解锁成就

**预置成就（16个，4类 × 4级稀有度）**：

| 分类 | 普通（common） | 稀有（rare） | 史诗（epic） | 传说（legendary） |
|------|---------------|-------------|-------------|------------------|
| **成长类** | 初识记忆 | 时光旅人 | 深海探索者 | 万象归一者 |
| **技能类** | 灵感火花 | 织梦者 | 炼金术士 | 创世之笔 |
| **社交类** | 初次相遇 | 温暖陪伴 | 心灵共鸣 | 灵魂挚友 |
| **特殊类** | 第一天 | 连续七天 | 赛季先锋 | 传奇守护者 |

**核心功能**：
- 成就列表查询（分类/状态筛选）
- 手动解锁 + 自动检测解锁
- 成就统计（总数/解锁数/按分类/按稀有度/解锁率）
- 解锁成就自动奖励天赋点数（普通1/稀有2/史诗3/传说5）
- 基于记忆数量、搜索次数、打卡天数的自动检测

**数据存储**：2 张表
- `growth_achievements` - 成就定义
- `growth_user_achievements` - 用户解锁状态

### 2.3 心智天赋树（talents）

**API 端点（5个）**：
- `GET /api/v1/growth/talents` - 获取天赋树（节点+连接+点数）
- `GET /api/v1/growth/talents/points` - 获取可用天赋点
- `POST /api/v1/growth/talents/{nodeId}/upgrade` - 升级天赋节点
- `POST /api/v1/growth/talents/reset` - 重置天赋树，返还点数
- `GET /api/v1/growth/talents/stats` - 天赋统计

**预置天赋（32个节点，4分支 × 3层结构）**：

| 分支 | Layer 1（基础，4节点） | Layer 2（进阶，3节点） | Layer 3（终极，1节点） |
|------|----------------------|---------------------|---------------------|
| **心智** | 逻辑萌芽 / 归纳直觉 / 演绎锋芒 / 假设检验 | 因果透镜 / 悖论耐受 / 系统思维 | 奥卡姆剃刀 |
| **稳态** | 情绪感知 / 呼吸锚定 / 认知重构 / 边界守护 | 压力转化 / 弹性心智 / 宁静致远 | 不动明王 |
| **创造** | 灵感捕捉 / 联想编织 / 跨界杂交 / 原型速构 | 美学直觉 / 逆向工程 / 实验精神 | 造物主 |
| **阅历** | 时间沉淀 / 模式识别 / 教训萃取 / 情境复现 | 历史透镜 / 相似联想 / 预见微光 | 万象归一 |

**核心功能**：
- 完整天赋树数据（节点 + 父子连接关系）
- 节点升级（点数检查 + 前置节点检查 + 满级检查）
- 重置返还所有点数
- 点数流水记录（来源/消耗/时间）
- 与成就系统联动（成就解锁奖励点数）
- 与赛季系统联动（任务完成奖励点数）

**数据存储**：3 张表
- `growth_talents` - 天赋定义
- `growth_user_talents` - 用户天赋状态
- `growth_points` - 点数变动流水

### 2.4 潮汐专属历法（calendar）

**API 端点（4个）**：
- `GET /api/v1/growth/calendar/{year}/{month}` - 月历数据
- `GET /api/v1/growth/calendar/day/{date}` - 单日数据
- `POST /api/v1/growth/calendar/checkin` - 打卡（心情+精力+备注+标签）
- `GET /api/v1/growth/calendar/stats` - 日历统计

**潮汐相位计算**：
- 基于 7天小潮 + 30天大潮 + 90天天文潮的正弦叠加模型
- 三档相位：小潮 / 大潮 / 天文潮
- 每日自动计算并标记

**核心功能**：
- 每日打卡（心情值 1-10 + 精力值 1-10 + 总结 + 标签）
- 月历视图（30天数据 + 潮汐相位标记）
- 统计汇总（总天数/打卡天数/连续天数/平均心情/平均精力）
- 打卡自动触发成就检测（第一天/连续七天等）
- 同一天重复打卡更新数据，不重复计数

**数据存储**：1 张表
- `growth_calendar` - 每日打卡数据

### 2.6 记忆回响对比（echo）

**API 端点（4个）**：
- `GET /api/v1/growth/memories` - 分页列表（page/size/category/keyword）
- `GET /api/v1/growth/memories/{id}` - 详情

- 分页查询 + 分类筛选

### 2.7 赛季征程系统（season）

**API 端点（5个）**：
- `GET /api/v1/growth/season/current` - 当前赛季详情
- `GET /api/v1/growth/season/history` - 历史赛季列表
- `GET /api/v1/growth/season/tasks` - 任务列表（type/phase_id 筛选）
- `POST /api/v1/growth/season/tasks/{taskId}/complete` - 完成任务
- `POST /api/v1/growth/season/tasks/{id}/claim` - 领取奖励

**预置赛季（2个）**：

| 赛季 | 状态 | 阶段数 | 任务数 |
|------|------|--------|--------|
| S1 启程之春 | 历史（已完成） | 6 | 24 |
| S2 觉醒之夏 | 当前（进行中） | 6 | 24 |

**6 阶段模板**：
1. 启程 — 建立基线
2. 深耕 — 聚焦主题
3. 突破 — 技能习得
4. 整合 — 体系构建
5. 验证 — 实践检验
6. 收官 — 沉淀归档

**核心功能**：
- 当前赛季详情（含阶段进度 + 剩余天数）
- 历史赛季回顾
- 任务列表（按类型/阶段筛选）
- 完成任务（获得天赋点）
- 领取阶段奖励（额外点数 + 限定成就）
- 与天赋系统联动（奖励发放到天赋点数）
- 自动计算进度百分比

**数据存储**：3 张表
- `growth_seasons` - 赛季
- `growth_season_phases` - 阶段
- `growth_season_tasks` - 任务

### 2.8 M8 代理路由（可选）

**新增文件**：`M8-control-tower/backend/routers/growth_m5_proxy.py`

26 个 API 端点的代理实现，将 `/api/growth/*` 转发到 M5 的 `/api/v1/growth/*`。

**特点**：
- 使用 httpx 直接发请求（不依赖 module_client 的 get_client 方法）
- 从环境变量读取 M5 地址和 Token
- 自动提取 M5 响应中的 data 字段
- 统一包装为 ApiResponse 格式

> 注：当前未挂载到 M8 主应用，保留 growth.py 本地实现。
> 如需启用，在 main.py 中将 growth_router 替换为 growth_m5_proxy_router。

## 三、模块间联动关系

```
┌─────────────────────────────────────────────────┐
│              成长系统（Growth System）             │
│                                                   │
│  赛季征程 ──完成任务──→ 天赋点 ←──解锁成就── 成就勋章 │
│     │                  ↑                  │       │
│     │                  │                  │       │
│     └──阶段奖励───────┘                  │       │
│                                        │       │
│  潮汐历法 ──打卡触发成就检测──────────→ 成就勋章 │
│                                                   │
│  地球Online编年史  ←── 数据来源 ──  记忆回响对比   │
│                                                   │
└─────────────────────────────────────────────────┘
         │
         │ 与核心记忆系统联动
         ▼
┌─────────────────────────────────────────────────┐
│            潮汐记忆核心（Tide Memory）             │
│                                                   │
│  记忆归档 → 数量统计 → 成就自动检测解锁             │
│  记忆检索 → 访问统计 → 成就自动检测解锁             │
│                                                   │
└─────────────────────────────────────────────────┘
```

**联动点汇总**：

| 联动方向 | 触发条件 | 效果 |
|---------|---------|------|
| 成就 → 天赋 | 成就解锁 | 奖励天赋点数（1/2/3/5点按稀有度） |
| 赛季 → 天赋 | 任务完成 / 阶段奖励 | 奖励天赋点数 |
| 历法 → 成就 | 打卡（第一天/连续七天） | 触发成就检测 |
| 记忆 → 成就 | 归档数量 / 检索次数 | 触发成长类成就检测 |
| 编年史 ↔ 回响 | 同类事件 | 数据可互通（预留） |

## 四、影响范围

### 4.1 新增能力

| 能力 | 说明 |
|------|------|
| 成就系统 | 16 个预置成就，自动检测解锁，奖励天赋点 |
| 天赋系统 | 32 个天赋节点，4 分支树形结构，点数升级 |
| 潮汐历法 | 每日打卡，心情/精力记录，潮汐相位计算 |
| 编年史 | 游戏化纪事管理，4 类事件分级 |
| 记忆回响 | before/after 对比，模板化成长感悟生成 |
| 赛季征程 | 180天赛季，6 阶段，任务驱动成长 |

### 4.2 兼容性

- **完全向后兼容**：所有核心记忆功能不受影响
- **纯新增功能**：游戏化模块为独立新增，不修改现有逻辑
- **数据隔离**：游戏化数据使用独立的数据库文件（growth.db）
- **可选启用**：前端页面可独立访问，不影响其他功能

### 4.3 风险评估

| 风险点 | 等级 | 应对措施 |
|--------|------|---------|
| 数据格式与前端不匹配 | 中 | 前端有 mock 数据降级，API 不兼容时自动回退 |
| 数值平衡问题 | 低 | 初始数值保守，后续可通过配置调整 |
| 性能影响 | 低 | 独立数据库，异步操作，不影响核心记忆路径 |
