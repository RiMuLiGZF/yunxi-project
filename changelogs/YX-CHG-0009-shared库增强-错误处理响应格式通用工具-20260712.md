## 一、背景

根据系统体检报告，shared 库存在以下问题：

- 两套实现并存（根目录版 + core/clients 分层版），结构混乱
- 缺少统一的错误处理体系，各模块自定义异常不统一
- 缺少标准 API 响应格式，各模块返回格式不一致
- 通用工具函数散落在各模块，重复实现

本次采取**渐进式增强**策略：
- 保留根目录稳定版作为主实现，不破坏现有功能
- 为实验性的 core/clients 子模块添加 DEPRECATED 标识
- 新增三个通用能力模块，供所有模块逐步复用

---

## 二、改动内容

### 2.1 废弃标识

为实验性子模块添加废弃说明：

| 文件 | 说明 |
|------|------|
| `shared/core/README.md` | 标注 core 子模块为实验性版本，推荐使用根目录稳定版 |
| `shared/clients/README.md` | 标注 clients 子模块为实验性版本，推荐使用根目录稳定版 |

### 2.2 新增 shared/errors.py - 统一错误处理

提供 7 个标准异常类 + 转换工具：

| 异常类 | 错误码 | 用途 |
|--------|--------|------|
| `YunxiError` | 50000 | 自定义异常基类 |
| `ConfigError` | 40002 | 配置错误 |
| `ModuleNotFoundError` | 40402 | 模块不存在 |
| `ModuleCallError` | 50302 | 模块调用失败 |
| `ValidationError` | 40001 | 参数验证错误 |
| `AuthenticationError` | 40101 | 认证错误 |
| `AuthorizationError` | 40301 | 授权错误 |

每个异常包含 `code`、`message`、`details` 三个属性，
提供 `error_to_dict()` 函数将异常转为字典（用于 API 响应）。

### 2.3 新增 shared/responses.py - 统一 API 响应格式

- **`ApiResponse` 类**：统一的 API 响应格式
  - `success(data, message, request_id)` - 成功响应工厂方法
  - `error(code, message, details, request_id)` - 错误响应工厂方法
  - `to_dict()` - 转为字典
  - `is_success` - 是否成功的属性

- **标准错误码常量**：

| 常量名 | 值 | 说明 |
|--------|-----|------|
| `SUCCESS` | 0 | 成功 |
| `ERROR_INVALID_PARAMS` | 40001 | 参数错误 |
| `ERROR_UNAUTHORIZED` | 40101 | 未认证 |
| `ERROR_FORBIDDEN` | 40301 | 无权限 |
| `ERROR_NOT_FOUND` | 40401 | 资源不存在 |
| `ERROR_INTERNAL` | 50001 | 内部错误 |
| `ERROR_MODULE_UNAVAILABLE` | 50301 | 模块不可用 |

### 2.4 新增 shared/utils.py - 通用工具函数

| 函数 | 功能 |
|------|------|
| `generate_id(length=16)` | 生成加密安全的随机十六进制 ID |
| `now_timestamp()` | 当前 Unix 时间戳（秒） |
| `now_iso()` | 当前 ISO 8601 格式时间字符串（UTC） |
| `safe_get(dict_obj, key, default)` | 安全获取字典值（支持 None 字典） |
| `truncate_text(text, max_length=100)` | 截断文本并添加省略号 |
| `format_file_size(bytes_size)` | 格式化文件大小（B/KB/MB/GB/TB） |

### 2.5 导出更新

- `shared/__init__.py` 新增 errors、responses、utils 三个模块的导出
- `shared/README.md` 同步更新文档，新增三个模块的说明和使用示例
