# YX-CHG-0002 — M3 端云协同修复 4 个 P0 功能性 Bug

- **编号**：YX-CHG-0002
- **日期**：2026-07-12
- **模块**：M3 端云协同
- **优先级**：P0
- **来源**：第一阶段修复（体检报告发现）

---

## 改动背景

体检报告发现 M3 端云协同存在大量 Mock 模式、API 功能失效等问题。
经深入检查，定位到 4 个明确的功能性 bug，导致核心组件初始化失败和 API 接口静默失效。

## 改动内容

**修改文件**：`M3-edge-cloud/server.py`

### Bug 1：DeviceRegistry 初始化变量名错误（NameError）

**位置**：第 157 行

修改前：
```python
_db_path = str(Path(_project_root) / "M3-edge-cloud" / "data" / "devices.db")
```

修改后：
```python
_db_path = str(Path(PROJECT_ROOT) / "M3-edge-cloud" / "data" / "devices.db") if PROJECT_ROOT else str(BASE_DIR / "data" / "devices.db")
```

**问题**：使用了未定义的 `_project_root` 变量，正确变量名是 `PROJECT_ROOT`。
**影响**：设备注册表组件始终落入 mock 模式。

---

### Bug 2：v1 冲突列表路由函数名错误（NameError）

**位置**：第 866 行

修改前：
```python
return await list_conflicts(request)
```

修改后：
```python
return await sync_conflicts(request, page=page, page_size=page_size)
```

**问题**：调用了不存在的 `list_conflicts()` 函数，实际函数名是 `sync_conflicts()`。
**影响**：调用 `/api/v1/sync/conflicts` 接口时直接报错。

---

### Bug 3：环境变量覆盖配置调用不存在的方法

**位置**：第 268-325 行（共 8 处）

修改前：
```python
try:
    config_manager.set("xxx", value)
except:
    pass
```

修改后：
```python
updates = {}
# 收集所有待更新项...
if updates:
    try:
        config_manager.update_config(updates)
    except:
        pass
```

**问题**：`ConfigManager` 类没有 `set()` 方法（只有 `get()` 和 `update_config()`），每次调用都被 try/except 吞掉异常。
**影响**：环境变量配置覆盖功能完全失效（端口、日志级别、CORS 等无法通过环境变量设置）。

---

### Bug 4：v1 别名 API 缺少 Query 参数声明

**位置**：第 855-870 行

修改前：
```python
@app.get("/api/v1/sync/conflicts")
async def v1_sync_conflicts(request: Request):
    return await sync_conflicts(request)
```

修改后：
```python
@app.get("/api/v1/sync/conflicts")
async def v1_sync_conflicts(
    request: Request,
    page: int = Query(1, ge=1),
    page_size: int = Query(20, ge=1, le=100)
):
    return await sync_conflicts(request, page=page, page_size=page_size)
```

**问题**：v1 别名函数直接调用 v3 处理函数时，`page`、`page_size` 等参数是 `Query` 对象而非 `int` 值，导致比较运算抛出 TypeError。
**影响**：`/api/v1/sync/conflicts` 和 `/api/v1/devices` 两个 v1 兼容接口实际走 mock 路径，返回空数据。

## 影响范围

| 组件正常初始化 | 7/8 | 8/8 |

| 环境变量覆盖功能 | 失效 | 正常 |

## 验证方式

1. 启动 M3 服务：`python server.py`
2. 检查启动日志，确认 8 个核心组件全部初始化成功（0 个 Mock）
3. 测试 `/api/v1/devices` 接口，确认返回真实数据而非空 mock
4. 测试 `/api/v1/sync/conflicts` 接口，确认不报错
5. 设置环境变量 `M3_PORT=8099` 启动，确认端口生效

## 回滚方案

仅修改了 `server.py` 一个文件，通过 git 回滚即可。
