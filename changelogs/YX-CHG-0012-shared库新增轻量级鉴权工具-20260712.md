## 一、背景

根据系统体检报告，云汐系统安全体系薄弱，各模块鉴权实现不统一：

- M11 有完整的数据库级鉴权方案，但太重，不适合小模块
- M6、M9 等模块没有鉴权或鉴权实现不一致
- 缺少轻量级、可复用的鉴权工具
- 速率限制实现分散，不统一

本次在 shared 库中新增**轻量级鉴权工具模块**，提供零依赖、易集成的鉴权能力，供所有模块按需复用。

---

## 二、改动内容

### 新增 `shared/auth.py`

包含 5 大功能模块：

| 功能 | 核心 API | 说明 |
|------|----------|------|
| **API Key 管理** | `hash_api_key()` / `verify_api_key()` | SHA256 哈希，支持明文密钥和（哈希+元数据）两种格式 |
| **路径白名单** | `is_public_path()` / `DEFAULT_PUBLIC_PATHS` | 通配符 `*` 匹配 + 前缀匹配，内置 7 条默认公开路径 |
| **令牌桶限流** | `SimpleRateLimiter` 类 | 线程安全的内存限流，支持 `check()` / `remaining()` / `reset()` |
| **FastAPI 适配** | `create_api_key_dependency()` | X-API-Key + Bearer 双模式，集成白名单和限流 |
| **实用工具** | `generate_api_key()` / `mask_api_key()` | 加密安全密钥生成 + 日志脱敏 |

### 2.1 API Key 管理

```python
from shared import hash_api_key, verify_api_key

# 生成密钥哈希
key_hash = hash_api_key("my-secret-key")

# 验证（支持明文列表）
result = verify_api_key("my-secret-key", ["key1", "key2"])

# 验证（支持哈希+元数据）
result = verify_api_key("my-secret-key", [
    ("key_hash_1", {"name": "admin", "permissions": ["*"]}),
])
```

### 2.2 路径白名单

```python
from shared import is_public_path, DEFAULT_PUBLIC_PATHS

# 默认公开路径：/, /health, /m8/*, /docs, /redoc, /openapi.json, /favicon.ico
is_public_path("/m8/health", DEFAULT_PUBLIC_PATHS)  # True
```

### 2.3 令牌桶限流

```python
from shared import SimpleRateLimiter

limiter = SimpleRateLimiter(default_limit=60, window_seconds=60)
allowed, remaining, window = limiter.check("user-123")
```

### 2.4 FastAPI 一键集成

```python
from shared import create_api_key_dependency

get_api_key = create_api_key_dependency(
    valid_keys=["my-secret-key"],
    rate_limiter=SimpleRateLimiter(),
    enabled=True,
)

@router.get("/protected")
async def protected(api_key = Depends(get_api_key)):
    return {"message": "ok"}
```

## 三、影响范围

### 3.1 新增能力
- 全模块统一的轻量级鉴权方案
- 标准的速率限制实现
- 统一的密钥生成和脱敏工具
- FastAPI 一键集成，接入成本极低

### 3.2 设计特点
- **零数据库依赖**：纯标准库实现，FastAPI 为可选延迟导入
- **线程安全**：`SimpleRateLimiter` 使用 `threading.Lock` 保证并发安全
- **易集成**：既提供纯函数接口，也提供 FastAPI `Depends` 一键集成
- **容错设计**：开发模式可一键关闭；短密钥全部隐藏不泄露

### 3.3 兼容性
- **完全向后兼容**：纯新增模块，不影响现有代码
- **可选使用**：各模块按需接入，不强制
- **渐进式**：M11 等已有完整鉴权的模块可继续使用原有方案
