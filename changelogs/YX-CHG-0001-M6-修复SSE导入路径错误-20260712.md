# YX-CHG-0001 — M6 修复 SSE 导入路径错误

- **编号**：YX-CHG-0001
- **日期**：2026-07-12
- **模块**：M6 硬件外设
- **优先级**：P0
- **来源**：第一阶段修复（体检报告发现）

---

## 改动背景

体检报告发现 M6 硬件外设模块存在 SSE 导入路径错误：
- `server.py` 中 `_get_m6_real_metrics()` 函数尝试从 `m6_hardware.services.sse_manager` 导入
- 实际文件位于 `m6_hardware/realtime/sse_manager.py`
- 会导致 `ImportError`，metrics 接口无法返回实时数据

## 改动内容

**文件**：`M6-hardware-peripheral/server.py` 第 234 行

修改前：
```python
from m6_hardware.services.sse_manager import get_sse_manager
```

修改后：
```python
from m6_hardware.realtime.sse_manager import get_sse_manager
```

与文件顶部的导入路径和实际文件位置保持一致。

## 影响范围

- 影响接口：`/m8/metrics`（调用 `_get_m6_real_metrics` 时）
- 影响功能：SSE 实时数据流相关的指标统计
- 风险等级：低（修复导入路径，无逻辑变更）

## 验证方式

1. 启动 M6 服务：`python server.py`
2. 访问 `http://localhost:8006/m8/metrics`
3. 确认不出现 ImportError，正常返回 metrics 数据

## 回滚方案

修改单行导入路径，如需回滚恢复原路径即可。
