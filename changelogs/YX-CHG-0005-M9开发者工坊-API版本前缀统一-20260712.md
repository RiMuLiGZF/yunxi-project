# YX-CHG-0005 M9 开发者工坊 API 版本前缀统一

| 项目 | 内容 |
|------|------|
| **编号** | YX-CHG-0005 |
| **模块** | M9 开发者工坊（M9-dev-workshop） |
| **改动类型** | 接口规范统一 |
| **日期** | 2026-07-12 |
| **优先级** | P0 |
| **关联体检报告** | YX-SYS-00 系统总览、YX-MOD-09 编程开发 |

---

## 一、背景

根据系统体检报告，云汐系统各模块 API 路径前缀不统一，存在以下问题：

- 部分模块使用 `/api/v1/` 前缀（如 M6、M7、M10、M11）
- 部分模块无版本号前缀（如 M8 中控塔、M9 开发者工坊）
- M3 同时存在 v1、v3 三套前缀
- M2 同时存在 v1、v2 两套前缀

M9 开发者工坊（`M9-dev-workshop`）的后端 API 全部使用无前缀的 `/api/xxx` 路径，与系统主流的 `/api/v1/` 规范不一致，增加了 API 网关统一管理和前端调用的集成成本。

---

## 二、改动内容

### 2.1 路由前缀统一

将 4 个路由模块的 prefix 从 `/api/xxx` 统一为 `/api/v1/xxx`：

| 路由模块 | 修改前前缀 | 修改后前缀 | 路由数量 |
|----------|-----------|-----------|---------|
| workspace | `/api/workspace` | `/api/v1/workspace` | ~15 |
| vscode | `/api/vscode` | `/api/v1/vscode` | ~12 |
| mcp | `/api/mcp` | `/api/v1/mcp` | ~10 |
| dashboard | `/api/dashboard` | `/api/v1/dashboard` | ~7 |

**涉及文件：**
- `routers/workspace.py`
- `routers/vscode.py`
- `routers/mcp.py`
- `routers/dashboard.py`

### 2.2 旧路径兼容重定向

在 `main.py` 中新增 `deprecated_api_redirect_middleware` 中间件，实现旧路径到新路径的平滑过渡：

**重定向规则：**
- 匹配路径：`/api/workspace/*`、`/api/vscode/*`、`/api/mcp/*`、`/api/dashboard/*`
- GET/HEAD 请求：返回 **308 Permanent Redirect**
- 其他方法（POST/PUT/DELETE 等）：返回 **307 Temporary Redirect**
- 完整保留查询参数和请求方法

**废弃标记响应头：**
- `Deprecation: true`（RFC 8594 标准）
- `Warning: 299 - "Deprecated API path: use /api/v1/ prefix instead"`
- `X-Deprecated-Reason: API path migrated to /api/v1/ prefix for unified versioning`

### 2.3 仪表盘快捷操作路径更新

更新 `dashboard.py` 中 `quick-actions` 接口返回的 action 路径：

| 操作 ID | 修改前路径 | 修改后路径 |
|---------|-----------|-----------|
| scan_projects | `/api/workspace/scan` | `/api/v1/workspace/scan` |
| launch_vscode | `/api/vscode/start` | `/api/v1/vscode/start` |
| mcp_tools | `/api/mcp/tools` | `/api/v1/mcp/tools` |
| new_project | `/api/workspace/projects` | `/api/v1/workspace/projects` |

---

## 三、影响范围

### 3.1 受影响模块
- M9 开发者工坊后端服务
- 调用 M9 API 的前端页面（仪表盘、工作区管理等）

### 3.2 兼容性
- **向后兼容**：旧路径通过重定向继续可用，不会导致现有调用方立即失效
- **推荐迁移**：调用方应尽快切换到 `/api/v1/` 前缀的新路径
- **废弃计划**：旧路径将在 v1.2.0 版本中正式移除（预计 3 个月后）

---

## 四、验证方法

### 4.1 新路径验证
```bash
# 测试工作区 API
curl http://localhost:8009/api/v1/workspace/projects

# 测试 VS Code API
curl http://localhost:8009/api/v1/vscode/status

# 测试 MCP API
curl http://localhost:8009/api/v1/mcp/tools

# 测试仪表盘 API
curl http://localhost:8009/api/v1/dashboard/overview
```

### 4.2 旧路径兼容验证
```bash
# 验证旧路径重定向（应返回 307/308）
curl -I http://localhost:8009/api/workspace/projects
# 预期：308 Permanent Redirect
# Location: /api/v1/workspace/projects
# Deprecation: true

# 验证 POST 请求重定向（应返回 307）
curl -I -X POST http://localhost:8009/api/workspace/scan
# 预期：307 Temporary Redirect
```

### 4.3 OpenAPI 文档验证
访问 `http://localhost:8009/docs`，确认所有 API 路径均显示为 `/api/v1/xxx` 格式。
