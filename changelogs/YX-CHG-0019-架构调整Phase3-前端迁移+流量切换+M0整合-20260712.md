## 一、变更背景

Phase 2 完成了 M4 后端业务模式代码迁移和 M8 代理网关建设，但存在以下遗留问题：
1. M4 前端页面不完整（缺少情绪陪伴、形象工坊 2 个模式页面）
2. M4 前端缺少 `colors_and_type.css` 样式文件，页面样式可能异常
3. M8 业务路由仍走本地实现，未接入 M4 代理（只有独立网关前缀）
4. M0 主理人管控台缺少 M4 业务模式管理入口

Phase 3 目标：补齐前端、打通流量、整合管控，实现三端协同的完整架构。

### 2.2 M8 业务流量切换（M4 代理中间件）

#### 2.2.1 新增文件

- `M8-control-tower/backend/m4_proxy_middleware.py` — M4 业务代理中间件

#### 2.2.2 三种代理模式

通过环境变量 `M4_BUSINESS_PROXY_MODE` 控制：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| `off` | 关闭代理，全部使用 M8 本地实现 | 默认值，稳定运行期 |
| `fallback` | 优先代理到 M4，M4 不可用时自动回退到本地 | 迁移过渡期、灰度验证 |
| `on` | 强制使用 M4 代理，M4 不可用时返回 503 | 全量切换后、M8 瘦身前 |

#### 2.2.3 代理的业务路径（11个）

| M8 路径前缀 | M4 路径前缀 | 说明 |
|------------|------------|------|
| `/api/growth/` | `/api/v1/growth/` | 成长中心 |
| `/api/work-dev/` | `/api/v1/work-dev/` | 工作开发 |
| `/api/review/` | `/api/v1/review/` | 复盘总结 |
| `/api/study-plan/` | `/api/v1/study-plan/` | 学业规划 |
| `/api/life-management/` | `/api/v1/life-management/` | 生活管理 |
| `/api/emotion-comfort/` | `/api/v1/emotion-comfort/` | 情绪陪伴 |
| `/api/social-relation/` | `/api/v1/social-relation/` | 人际关系 |
| `/api/appearance/` | `/api/v1/appearance/` | 形象工坊 |
| `/api/chat/` | `/api/v1/chat/` | 聊天服务 |
| `/api/voice/` | `/api/v1/voice/` | 语音服务 |
| `/api/watch/` | `/api/v1/watch/` | 手表交互 |

#### 2.2.4 中间件特性

- **透明代理**：请求路径不变，对前端无感
- **请求透传**：完整透传请求头、查询参数、请求体
- **响应标识**：添加 `X-M4-Proxied: true` 响应头标识代理来源
- **错误分级**：连接失败 → 503，超时 → 回退，其他 → 502
- **性能开销**：中间件层仅做路径匹配和转发，开销 < 1ms

#### 2.2.5 运行时动态切换

在 M8 的 m4_gateway 路由中新增了运行时切换接口：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/m4-gateway/business-proxy/status` | GET | 查询当前代理模式 |
| `/api/m4-gateway/business-proxy/mode` | POST | 动态切换代理模式 |

> 注意：运行时切换仅影响当前进程，重启后恢复为环境变量配置值。

## 三、技术亮点

1. **三档流量切换** — off/fallback/on 三档渐进式切换，风险可控
2. **中间件层代理** — 在 FastAPI 中间件层实现透明代理，无需逐个修改路由
3. **运行时动态切换** — 支持通过 API 动态切换代理模式，无需重启服务
4. **向后兼容** — 默认模式为 off，不影响现有任何功能
5. **三端协同** — M0（管控） + M4（业务） + M8（技术） 架构初步成型
