# V9.7→V9.9 优化对比清单

> 版本对比：V9.7 vs V9.9
> 迭代轮次：2轮增量优化（V9.8 第一轮3项创新 + V9.9 第二轮2项修复）
> 测试覆盖：558项测试全量通过（V9.7: 541 → V9.9: 558，+17）
> 日期：2026-06-28

---

## 一、修复总览表

| 维度 | V9.7 状态 | V9.9 状态 | 提升 |
|------|----------|----------|------|
| 任务取消能力 | TaskDispatcher已集成CancelToken，但OrchestratorV9未暴露取消API | `OrchestratorV9.__getattr__`白名单新增`cancel_task`，上层可直接调用 | 端到端取消链路打通 |
| 语义路由精度 | 字符级n-gram嵌入，无IDF加权 | 新增TF-IDF加权：稀有n-gram获得更高权重，自动重建IDF表 | 路由区分度显著提升 |
| OOM优雅降级 | 无降级机制，模型请求超显存时直接失败 | `ModelRotationManager`三级降级链（gpt-4o→gpt-4o-mini→mock-model）+ 降级事件日志 | 可用性大幅提升 |
| 任务取消透传 | 无 | `TaskDispatcher._execute_single()`通过inspect检测Agent是否支持`cancel_token`参数，支持则透传 | 协作式取消落地 |
| 语义路由 Agent变动 | n-gram频率不变，Agent注册/注销不影响其他Agent权重 | Agent注册/注销触发`_rebuild_idf()`，全局IDF表自动重建 | 动态适应性增强 |

## 四、性能影响对比

| 操作 | V9.7 | V9.9 | 改善 |
|------|------|------|------|
| 任务取消延迟 | 不支持 | <1ms（Event信号） | 新增能力 |
| 语义路由计算 | O(N) n-gram频率 | O(N) TF-IDF加权（+IDF查表开销） | 精度提升，开销可忽略 |
| IDF重建 | 无 | O(M) M为所有Agent描述总字符数 | Agent变动时一次性开销 |
| OOM模型降级 | 直接失败 | 遍历降级链（最多3次检测） | 可用性提升 |
| 模型释放 | 手动 | 切换模式自动释放 | 减少显存泄漏 |
| CancelToken透传 | 无 | inspect检查 + kwargs透传（~0.05ms） | 协作式取消支持 |

**SemanticRouter TF-IDF加权效果示例**：

```
场景：3个Agent —— code_agent（"编写Python代码"）、writing_agent（"撰写文章"）、math_agent（"数学计算"）

V9.7无IDF：
  任务"帮我写个排序算法"
  → code_agent和math_agent都可能因包含"算"字获得相似权重
  → 区分度不足

V9.9有IDF：
  "代码"在code_agent描述中为高频特有词 → 低IDF但高TF → 适中权重
  "Python"仅在code_agent出现 → 高IDF → 显著放大code_agent得分
  "撰写"仅在writing_agent出现 → 高IDF → writing_agent得分被相对抑制
  → code_agent排名显著领先
```

**OOM降级链效果示例**：

```
场景：max_vram=3000MB，请求gpt-4o（5000MB）

V9.7：直接失败，返回None或抛出异常

V9.9：
  1. 检测gpt-4o（5000MB > 3000MB）→ 超显存
  2. 释放当前模型（如有）
  3. 尝试gpt-4o-mini（1000MB <= 3000MB）→ 成功
  4. 记录降级日志：{"requested": "gpt-4o", "allocated": "gpt-4o-mini", "reason": "oom_fallback"}
  5. 返回"gpt-4o-mini"
```

## 五、专利创新点演进

### V9.7 专利创新点（5项）

| 编号 | 创新点 | 技术方案 | 局限 |
|------|--------|---------|------|
| 1 | Safe-Ledger双层防死循环 | plan级5次+任务级20次+SKIPPED终态+close_task闭环 | 稳定，无变化 |
| 2 | TLS-Vis三层分级可见性 | guest/member/owner动态过滤 + 动作词标记 | 稳定，无变化 |
| 3 | Heuristic-Stall-Detector | 零进展/循环依赖/孤儿计划纯规则检测 | 稳定，无变化 |
| 4 | Adaptive-Retry自适应重试 | 五类错误分类 + EMA动态学习 + 混合延迟 | 稳定，无变化 |
| 5 | A2A-Task-Signed端到端签名 | Task级HMAC-SHA256 + HTTPTransport自动集成 | 稳定，无变化 |

### V9.9 专利创新点（6项，+1项新增）

| 编号 | 创新点 | 演进路径 | V9.9强化内容 |
|------|--------|---------|-------------|
| 1 | Safe-Ledger双层防死循环 | V9.5基础→V9.6 close_task闭环→V9.9稳定 | 维持V9.7设计 |
| 2 | TLS-Vis三层分级可见性 | V9.5稳定→V9.9无变化 | 维持V9.7设计 |
| 3 | Heuristic-Stall-Detector | V9.5稳定→V9.9无变化 | 维持V9.7设计 |
| 4 | Adaptive-Retry自适应重试 | V9.5基础→V9.6错误分类→V9.7 EMA学习→V9.9稳定 | 维持V9.7设计 |
| 5 | A2A-Task-Signed端到端签名 | V9.5仅AgentCard→V9.6 Task签名API→V9.7 HTTPTransport集成→V9.9稳定 | 维持V9.7设计 |
| 6 | **SemanticRouter轻量语义路由+TF-IDF加权** | V9.8基础n-gram路由 → **V9.9 TF-IDF加权** | 新增IDF加权，稀有n-gram获得更高权重，路由精度显著提升 |

**创新点6演进详解**：

```
V9.7: 无SemanticRouter，Agent路由依赖Registry的capability硬编码匹配或LLM-based路由

V9.8: 引入SemanticRouter基础版：
  - 字符级n-gram嵌入（默认n=2）
  - L2归一化 + 余弦相似度
  - 纯本地，无LLM调用
  - 局限：仅使用n-gram频率，无区分度加权

V9.9: 引入TF-IDF加权增强：
  - _rebuild_idf(): 基于所有Agent描述重建IDF表
  - idf(g) = log((doc_count + 1) / (freq + 1)) + 1.0
  - 嵌入计算：weighted = {g: freq * idf(g) for g, freq in grams.items()}
  - Agent注册/注销时自动重建IDF表
  - 效果：特有术语（如"Python"、"排序算法"）获得更高区分权重
```
