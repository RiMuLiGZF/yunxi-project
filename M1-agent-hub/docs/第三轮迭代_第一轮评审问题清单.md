## 遗留问题（第二轮处理）

### P3-001 [P1] TaskDispatcher CircuitBreaker 双段式集成
**问题**：if检查open + breaker.call()两段式存在竞态和逻辑冗余。
**修复**：删除前置if，统一使用breaker.call()包裹。

### P3-002 [P1] retry_coordinator 与 task_dispatcher 仍未实际联动
**问题**：TaskDispatcher._execute_with_retry()硬编码重试1次，未接入RetryCoordinator。
**修复**：TaskDispatcher注入RetryCoordinator，_execute_with_retry()调用coordinator.check_can_retry()。

### P3-003 [P1] MessageAdapter 未在 app_bootstrap 中使用
**问题**：MessageAdapter实现了双向转换，但bootstrap未实例化，BusMessage↔A2A Task仍是物理割裂。
**修复**：bootstrap中实例化MessageAdapter并注册到bus和transport。

### P3-004 [P1] process_stream() 完全跳过风控
**问题**：process_stream()直接透传，无Guardrails/Budget/Ledger。
**修复**：process_stream()入口增加Guardrails检查，流式过程中插入预算增量检查。

### P3-005 [P2] 简单查询场景过度执行
**问题**："hello"等简单查询仍执行完整V3+Guardrails+Ledger。
**修复**：引入查询复杂度预筛，简单查询跳过V3和Ledger。
