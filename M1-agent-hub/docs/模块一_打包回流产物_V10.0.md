## 一、模块完整架构说明文档

### 1.1 核心定位

模块一（M1）是云汐系统的**多 Agent 集群调度中枢**，负责：
- 全局任务编排与生命周期管理
- Agent 注册发现与负载均衡
- 任务分发与硬件感知调度
- A2A 跨 Agent 通信协议
- 输入风控（Guardrails）与预算管控
- 双层任务账本（Ledger）死循环防护

### 1.2 七层架构（V2→V9）与 V10.0 整改

```
V9 OrchestratorV9          [编排入口]  Guardrails + Ledger + Budget + Intent + GroupChat
  ↓
V8 OrchestratorV8          [执行委托]  Pass-through + Tracer + TraceToMemory
  ↓
V7 OrchestratorV7          [调度增强]  Ensemble + Budget-aware + Retry
  ↓
V5 OrchestratorV5          [扩展能力]  SemanticRouter + GuardrailsV2 + LoopGuard
  ↓
V4 OrchestratorV4          [事件驱动]  Streaming + A2A + LLMProvider
  ↓
V3 OrchestratorV3          [语义路由]  IntentClassification + ClassificationRouter
  ↓
V2 OrchestratorV2          [基础执行]  AgentRegistry + TaskDispatcher + MessageBus
```

**[V10.0-R08] 架构简化评估：**
- V5层越界组件（VectorMemory/PluginLoader/MCPServer）已标记DEPRECATED，待迁移至模块2/5
- LLMProvider推理执行已标记DEPRECATED，待迁移至模块3
- 建议V11.0将七层合并为三层（编排决策层/调度执行层/扩展适配层）

#### api/server.py（FastAPI封装层）

```python
def create_server(orchestrator, registry, ledger, message_bus, health_monitor=None):
    app = FastAPI(title="云汐内核 API V10.0", version="10.0.0")

    @app.post("/api/v1/tasks/submit")
    async def submit_task(request: SubmitTaskRequest):
        # [V10.0-R05] M1唯一入口，解决M3命名冲突
        result = await orchestrator.process(user_input=request.user_input, ...)
        return JSONResponse(content={"status": result.get("status"), ...})

    @app.delete("/api/v1/agents/{agent_id}")
    async def delete_agent(agent_id: str):
        # [V10.0-R04] 补全缺失接口
        await registry.unregister(agent_id)
        return JSONResponse(content={"status": "success", "action": "unregistered"})

    @app.get("/api/v1/agents/{agent_id}/status")
    async def get_agent_status(agent_id: str):
        status = await registry.get_status(agent_id)
        if status is None: raise HTTPException(status_code=404, ...)
        return JSONResponse(content=status)

    @app.get("/api/v1/tasks/{task_id}/status")
    async def get_task_status(task_id: str):
        status = ledger.query_task(task_id)
        if status is None: raise HTTPException(status_code=404, ...)
        return JSONResponse(content=status)

    @app.post("/api/v1/bus/publish")
    async def bus_publish(request: BusPublishRequest):
        msg = BusMessage(topic=request.topic, ...)
        await message_bus.publish(msg)
        return JSONResponse(content={"status": "published", ...})
```

#### task_dispatcher.py（硬件感知调度）

```python
class TaskDispatcher:
    def __init__(self, registry, message_bus, ...):
        # [V10.0-R06] 硬件状态映射与断连缓存
        self._hardware_status: dict[str, Any] = {}
        self._offline_cache: dict[str, list[Any]] = {}

    def update_hardware_status(self, status: Any) -> None:
        device_id = getattr(status, "device_id", "")
        prev = self._hardware_status.get(device_id)
        was_online = getattr(prev, "online", True) if prev else True
        self._hardware_status[device_id] = status
        is_online = getattr(status, "online", True)
        if not was_online and is_online:
            asyncio.create_task(self._flush_offline_cache(device_id))

    def is_hardware_online(self, device_id: str) -> bool:
        status = self._hardware_status.get(device_id)
        return getattr(status, "online", True) if status else True

    async def dispatch(self, task: AgentTask) -> AgentResult:
        # [V10.0-R06] 硬件状态检查
        device_id = task.metadata.get("device_id", "")
        if device_id and not self.is_hardware_online(device_id):
            device_type = getattr(self._hardware_status.get(device_id), "device_type", "")
            if device_type == "drone":
                return AgentResult(task_id=task.task_id, status="failure", error=f"Drone offline")
            self._offline_cache.setdefault(device_id, []).append(task)
            return AgentResult(task_id=task.task_id, status="partial", error="Device offline, task cached")
        # ... 原有分发逻辑
```
