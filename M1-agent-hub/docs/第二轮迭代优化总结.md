# 第二轮 · 迭代优化总结

## 本轮补充调研内容

针对第一轮遗留的P1问题，重点调研：
- **Prompt Injection 攻防技术**：规则检测的局限性、对抗性变体（base64/Unicode/零宽字符）、语义级检测方案
- **Magentic-One Ledger 设计模式**：Task Ledger（外层计划循环）与 Progress Ledger（内层执行循环）的数据结构、偏差检测算法、重规划触发条件
- **对话收敛检测**：基于Embedding的余弦相似度 vs 基于TF-IDF的轻量方案、滑动窗口策略、多语言适配

## 本轮架构补充设计

在V9架构基础上补充3个治理层组件：
1. **Guardrails V2**：从简单关键词过滤升级为规则+语义组合+PII脱敏的三层安检
2. **Ledger Engine**：引入Magentic-One式双层账本，实现复杂任务的自校正循环
3. **ConvergenceTermination**：GroupChat的对话收敛检测，避免无意义空转

## 本轮评审结果

输出《第二轮评审问题清单》：
- 新增功能剩余缺陷：8项（对抗性变体检测、Ledger持久化、Convergence IDF权重等）
