# V9.4→V9.5 优化对比清单

> 版本对比：V9.4 vs V9.5
> 迭代轮次：2轮增量优化（第一轮7项修复 + 第二轮6项修复）
> 测试覆盖：526项测试全量通过
> 日期：2026-06-28

---

## 修复总览

| 维度 | V9.4 状态 | V9.5 状态 | 提升 |
|------|----------|----------|------|
| 死循环防护 | 无 replan 保护，可无限循环 | plan级5次 + 任务级20次 + SKIPPED终态 | 关键修复 |
| 属性穿透 | 全量透传所有 V8 属性 | 白名单10方法 + 可扩展 | 安全提升 |
| 预算前置检查 | 仅 V9/V8 入口检查 | 全链路 V9→V8→V7→Dispatcher | 预算覆盖100% |
| A2A桥接 | outbound handler 无人消费 | MemoryTransport 直接调用 handlers | 数据流打通 |
| HTTP订阅 | 空实现（yield一次退出） | 轮询 + 指数退避 + 可配置 | 功能完整 |
| 预算存储 | list 无限增长 + O(N) 检查 | deque(100K) + O(1) rolling | 性能10-100x |
| 访客可见性 | guest 完全看不到 agent 消息 | 完整user消息 + agent摘要 + 动作词标记 | 协作可用 |
| 白名单维护 | frozenset 硬编码 | set + register_passthrough() | 可扩展 |
| replan清理 | 无 TTL，内存泄漏 | 每小时清理 stale 条目 | 内存安全 |
| 进度评估 | 仅依赖状态检测 | +零进展 + 循环依赖启发式 | 智能化 |

## 性能影响

| 操作 | V9.4 | V9.5 | 改善 |
|------|------|------|------|
| check_budget（同日） | O(N) 全量扫描 | O(1) 增量读取 | ~100x @ 10K条记录 |
| check_budget（跨日） | O(N) | O(N)（首次）→ O(1)（后续） | 摊销 O(1) |
| \_\_getattr\_\_ 访问 | O(1) 但暴露全部 | O(1) 仅白名单 | 安全性 ↑ |
| dispatch（预算不足） | 执行后才拦截 | 执行前拦截 | 节省1次 Agent 调用 |
| subscribe（无服务） | 立即返回 | 轮询至超时 | 功能可用 |
