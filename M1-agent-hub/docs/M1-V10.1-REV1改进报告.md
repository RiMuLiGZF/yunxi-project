## 2. 命名映射表（底层模式 ↔ 上层场景 ↔ 调度策略）

### 2.1 底层模式 ↔ 上层场景（一一对应主映射）

| L1 底层模式（M4标准） | L2 上层场景（用户可见） | 中文名 |
|----------------------|----------------------|--------|
| `DOCUMENT` | `study_plan` | 学业规划 ↔ 文档写作 |
| `CODING` | `work_dev` | 工作开发 ↔ 代码开发 |
| `REVIEW` | `review_summary` | 复盘总结 ↔ 评审复盘 |
| `DESIGN` | `relationship` | 人际关系 ↔ 设计规划 |
| `MENTAL` | `emotion_companion` | 情绪陪伴 ↔ 情绪支持 |
| `PLANNING` | `life_management` | 生活综合管理 ↔ 计划管理 |

### 2.2 M1 调度策略（独立命名空间）

| 调度策略编号 | 中文名 | 说明 |
|------------|--------|------|
| `STRAT-A` | 简单任务直调 | 单意图、无依赖、低复杂度 |
| `STRAT-B` | 复杂任务DAG编排 | 多步骤、有依赖、中/高复杂度 |
| `STRAT-C` | 端云协同计算 | 端云资源选择或网络状态变化 |
| `STRAT-D` | 涉密内容处理 | 命中涉密关键词或显式标记分级 |
| `STRAT-E` | 多Agent冲突仲裁 | 死锁、资源争用或循环依赖 |
| `STRAT-F` | 断点续跑恢复 | 系统重启、Agent异常退出 |

### 2.3 命名空间边界说明

- **M4 业务场景层**：决定「做什么类型的事」，6种底层执行模式
- **M1 调度策略层**：决定「用什么方式组队执行」，6种调度策略 STRAT-A~F
- **两者正交**：同一业务场景可能触发不同调度策略（如 CODING 场景下，简单任务走 STRAT-A，复杂任务走 STRAT-B）
