## 新增功能评审

### ✅ Guardrails V2

**已实现**：
- Prompt Injection 规则检测（8种攻击模式，中英双语）
- 语义组合风险启发式评分
- PII 实体检测与脱敏（手机号/身份证/银行卡/邮箱/IP）
- 与 OrchestratorV9.process() 入口集成

**剩余缺陷**：
1. **对抗性变体检测空白**：未覆盖 base64 编码注入、Unicode 同形字符（如 Cyrillic а 替代 Latin a）、零宽字符绕过等高级攻击手法
2. **缺少动态阈值**：固定 threshold=0.7，未根据历史误报率自适应调整
3. **PII 正则在中国大陆场景下可靠，但国际场景覆盖不足**（如护照号、社保号等）

### ✅ Ledger Engine

**已实现**：
- Task Ledger：计划项管理、依赖解析、完成率计算、阻塞检测
- Progress Ledger：Agent 进度追踪、偏差事件记录、超时停滞检测
- 重规划评估：4 种触发条件（blockers / stalled / too_many_deviations / progress_stalled）
- 与 OrchestratorV9.process() 浅层集成

**剩余缺陷**：
4. **持久化缺失**：Ledger 纯内存存储，进程崩溃后全部丢失，未对接 SQLite/Checkpointer
5. **计划生成依赖外部**：TaskPlan 需外部传入，未实现基于意图的自动任务分解
6. **Orchestrator 集成浅层**：process() 仅在入口处创建 Ledger 记录，Agent 执行完成后未自动更新 Progress Ledger，偏差检测未闭环

### ✅ ConvergenceTermination

**已实现**：
- 基于 TF 词频向量的对话收敛检测
- 滑动窗口 + 平均相似度判定
- 与 GroupChat CompositeTermination 组合生效

**剩余缺陷**：
7. **未使用 IDF 权重**：仅 term frequency，高频词（"的"/"the"/"这个"）主导相似度，可能误判
8. **阈值硬编码**：similarity_threshold=0.85 缺乏任务类型适配（技术讨论 vs 闲聊应有不同阈值）
