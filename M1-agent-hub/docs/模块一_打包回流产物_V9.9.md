# 1 新版模块架构说明文档

## 1.1 模块核心定位与职责边界

**模块1：多Agent集群调度架构** 是云汐项目的中央调度中枢，负责：
- 全局任务编排与生命周期管理
- Agent注册发现、动态组队、负载均衡
- A2A跨Agent通信协议适配
- 输入风控安检、预算管控、死循环防护
- 任务账本跟踪与自动重规划

**严格边界**：仅处理调度逻辑，不实现记忆存储（模块5）、技能工具（模块2）、端云路由（模块3）、业务场景编排（模块4）、硬件驱动（模块6）。通过标准化接口与上述模块交互。

## 1.3 内部数据流流转逻辑

```
用户输入
  → GuardrailsV2安检（PromptInjection+PII）
  → 简单查询预筛（<20字直接透传V8）
  → SemanticIntentV3分类 → override_intent
  → BudgetManager预算检查
  → V8→V7→V5→V4→V3→V2 委托链
    → V2: MasterScheduler调度
    → TaskDispatcher分发（含CancelToken+CircuitBreaker+RetryCoordinator）
    → Agent.handle_task执行
  → AgentResult返回
  → LedgerEngine.close_task归档
  → 结果返回用户
```

## 1.4 启动执行流程

1. `ConfigManager` 加载配置
2. `MessageBus` 单例启动
3. `MemoryTransport` + `MessageAdapter` 双向桥接注册
4. 逐层构建 V2→V3→V4→V5→V7→V8→V9
5. 条件懒加载：未启用组件不实例化
6. 注册生命周期钩子（startup/shutdown）
7. 注册健康检查
8. `OrchestratorV9` 作为统一入口暴露

## 1.5 硬件联动逻辑

- **手表/戒指**：通过 `APIServer` HTTP接口接收精简指令，TaskDispatcher设置短TTL（30s），StreamingEngine按需流式返回
- **桌面屏**：完整V9流程，启用Guardrails+Ledger全链路跟踪，GroupChat支持多Agent协作
- **无人机**：高优先级任务（priority=1）通过TaskDispatcher抢占式调度，CancelToken支持紧急中断，BudgetManager限制单次请求成本

from pydantic import BaseModel, Field

class AgentTask(BaseModel):
    task_id: str = Field(default_factory=lambda: uuid.uuid4().hex)

    target: str = ""
    intent: str = ""
    payload: dict[str, Any] = Field(default_factory=dict)

class AgentResult(BaseModel):
    task_id: str = ""
    status: Literal["success", "failure", "partial", "timeout", "handoff"] = "success"
    error: str | None = None
    latency_ms: float = 0.0

class BusMessage(BaseModel):
    msg_id: str = Field(default_factory=lambda: uuid.uuid4().hex)

### orchestrator_v9.py - 统一编排入口
```python
class OrchestratorV9:
    _V8_PASS_THROUGH: set[str] = {
        "load_plugins", "get_config", "list_agents", "get_agent",
        "process_ensemble", "process_budget_aware", "acquire_model",
        "route_by_load", "filter_memories", "process_with_swarm",
        "cancel_task",
    }

    async def process(self, user_input: str, **kwargs: Any) -> dict[str, Any]:
        # 1. Guardrails安检
        guard_result = self._run_guardrails(user_input)
        if guard_result.blocked:
            return {"status": "blocked", "reason": guard_result.block_reason}
        # 2. 简单查询预筛
        if self._is_simple_query(user_input):
            return await self._delegate_v8(guard_result.sanitized_text, kwargs)
        # 3. Ledger跟踪
        task_id = kwargs.get("task_id", "")
        if task_id:
            self._ledger.create_task(task_id=task_id, goal=user_input)
        # 4. V3意图分类注入
        kwargs["override_intent"] = self._run_intent(guard_result.sanitized_text)
        # 5. Budget检查
        if not self._run_budget(task_id, ...):
            return {"status": "budget_exceeded"}
        # 6. V8委托 + Ledger闭环
        result = await self._delegate_v8(guard_result.sanitized_text, kwargs)
        return result
```

### task_dispatcher.py - 任务分发器
```python
class TaskDispatcher:
    def __init__(self, registry, message_bus, circuit_breaker=None,
                 retry_coordinator=None, budget_manager=None):
        self._registry = registry
        self._bus = message_bus
        self._circuit_breaker = circuit_breaker
        self._retry_coordinator = retry_coordinator
        self._budget_manager = budget_manager
        self._cancel_tokens: dict[str, CancelToken] = {}  # [V9.8]

    async def dispatch(self, task: AgentTask) -> AgentResult:
        # [V9.5] Budget预检
        if not self._check_budget(task):
            return AgentResult(status="failure", error="Budget exceeded")
        # [V9.8] 注册取消令牌
        self._cancel_tokens[task.task_id] = CancelToken()
        # 查找Agent → 执行 → 发布完成事件
        result = await self._execute_with_retry(task, agent, trace_id)
        self._cancel_tokens.pop(task.task_id, None)
        return result

    def cancel_task(self, task_id: str, reason: str = "") -> bool:
        token = self._cancel_tokens.get(task_id)
        if token:
            token.cancel(reason)
            return True
        return False
```

### semantic_router.py - 语义路由
```python
class SemanticRouter:
    def __init__(self, n: int = 2, use_idf: bool = True):
        self._capability_index: dict[str, set[str]] = {}  # [V9.6] O(1)
    def register_agent(self, agent_id: str, desc: str) -> None:
        self._agent_profiles[agent_id] = desc
        self._agent_embeddings[agent_id] = self._embed(desc)
        if self._use_idf:
            self._rebuild_idf()
    def route(self, task_description: str, top_k: int = 1) -> list[tuple[str, float]]:
        task_emb = self._embed(task_description)
        scores = [(aid, self._cosine_similarity(task_emb, emb))
                  for aid, emb in self._agent_embeddings.items()]
        scores.sort(key=lambda x: x[1], reverse=True)
        return scores[:top_k]
```

## 关键通信协议规范

**A2A Protocol v1.0**
- Task状态机: submitted → working → input_required → completed/failed/cancelled
- 消息格式: JSON-RPC 2.0 (`{"jsonrpc":"2.0","method":"tasks/send","params":{...},"id":"..."}`)
- 签名: HMAC-SHA256(task_id+sender+recipient+payload+created_at), secret前缀32位
- 传输绑定: MemoryTransport(内部) + HTTPTransport(外部轮询)

**MCP通信**
- 通过MCPServer暴露AgentRegistry和TaskDispatcher能力
- 模块2通过SkillsInterface调用本模块的Agent调度能力

**本地7B模型调用**
- 通过LLMProviderFactory.create(provider_type, model)创建推理实例
- ModelRotationManager管理显存+降级链，适配Q4_K_M量化模型
- BudgetManager按token计费，支持gpt-4o/gpt-4o-mini/mock-model定价
