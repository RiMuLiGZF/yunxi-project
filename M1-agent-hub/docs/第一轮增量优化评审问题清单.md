## 新发现的遗留问题

### P3-001 [严重] override_intent 逐层透传是"补丁式"修复，非架构级解法

**问题**：V9→V8→V7→V5→V4→V3→V2 共6层方法签名均需修改以传递 `override_intent`。这种模式维护成本高，新增一层（V10）需再次改动所有签名。真正的架构解法是在 Bootstrap 阶段将 `SharedContext`（包含tracer、classifier、registry等）组装为独立对象，所有层共享此对象。

**优先级**：中（当前修复可用，但技术债务仍在）

### P3-002 [严重] BusMessage 与 A2A Task 仍未统一（P2-006未修复）

**问题**：两套消息模型完全割裂的问题未被触碰。TaskDispatcher 仍使用 BusMessage 发布事件，A2AClient 使用 A2A Task。跨进程通信场景下两者无法互操作。

**优先级**：高

### P3-003 [中等] GroupChat 无 RBAC 约束（P2-013未修复）

**问题**：GroupChat 中的 Agent 不受 RBAC 权限约束，guest 角色可读取所有对话历史中的敏感信息。

**优先级**：高

### P3-004 [中等] Trace-to-Memory 数据流仍断裂（P2-016未修复）

**问题**：TraceToMemory 提取结果仅记录数量，未通过 MemoryInterface 写入任何记忆系统。

**优先级**：高

### P3-005 [中等] BudgetManager 在标准 process() 链中仍无预算检查点（P1-010/P2-004未修复）

**问题**：override_intent 修复后，标准处理链的预算检查仍被绕过。仅在 process_budget_aware() 中可选调用。

**优先级**：中

### P3-006 [低] LazyAgentRegistry 未被集成（P2-021未修复）

**问题**：启动时仍全量创建25+组件。

**优先级**：低（7B场景可优化，不影响功能）

### P3-007 [低] V5 中仍有多处 `self._v4._v3._v2` 深层穿透

**问题**：orchestrator_v5.py 中约5处深层属性链访问未消除。

**优先级**：低（功能正确，仅代码质量问题）
