# 三级域权限设计

> ⚠️ 本规范为M5潮汐记忆系统内部安全规范

## 域层级结构

```
┌─────────────────────────────────────┐
│         全局核心域 (core)            │
│  系统级记忆、配置、规则（只读）       │
└─────────────────────────────────────┘
         ▲              ▲
         │共享           │共享
┌─────────────────┐ ┌─────────────────┐
│  协作共享域A     │ │  协作共享域B     │
│  团队/项目记忆   │ │  团队/项目记忆   │
└─────────────────┘ └─────────────────┘
         ▲              ▲
         │私有           │私有
┌─────────────────┐ ┌─────────────────┐
│  Agent1私有域   │ │  Agent2私有域   │
│  个人记忆       │ │  个人记忆       │
└─────────────────┘ └─────────────────┘
```

## 域定义

### 1. 私有域 (private)

- **范围**：每个Agent独立拥有
- **所有者**：创建该记忆的Agent
- **访问权限**：
  - 所有者：读/写/删除
  - 其他Agent：无权限
- **用途**：个人记忆、偏好、私密对话

### 2. 共享域 (shared)

- **范围**：多个Agent共享
- **所有者**：创建域的Agent
- **访问权限**：
  - 管理员：读/写/删除/管理权限
  - 成员：读/写
  - 非成员：只读（可配置）
- **用途**：团队协作、项目共享

### 3. 核心域 (core)

- **范围**：全局系统级
- **所有者**：系统
- **访问权限**：
  - 系统：读/写
  - 核心Agent：只读
  - 普通Agent：无权限
- **用途**：系统规则、全局配置、核心知识库

## 权限矩阵

| 操作 | 私有域（所有者） | 私有域（他人） | 共享域（成员） | 共享域（非成员） | 核心域 |
|------|-----------------|---------------|---------------|-----------------|--------|
| 读取 | ✅ | ❌ | ✅ | ⚠️ 只读 | ⚠️ 只读 |
| 写入 | ✅ | ❌ | ✅ | ❌ | ❌ |
| 删除 | ✅ | ❌ | ⚠️ 仅自己 | ❌ | ❌ |
| 管理权限 | ✅ | ❌ | ⚠️ 管理员 | ❌ | ❌ |

## 权限检查流程

```
请求到达
   ↓
身份认证（JWT Token）
   ↓
提取Agent ID和角色
   ↓
判断目标域类型
   ↓
┌──────────┬──────────┬──────────┐
│ 私有域   │ 共享域   │ 核心域   │
└──────────┴──────────┴──────────┘
   ↓          ↓          ↓
检查所有权  检查ACL    检查角色
   ↓          ↓          ↓
权限决策 → 审计记录 → 返回结果
```

## API 接口

### 检查权限

```python
from tide_memory.security.domain_manager import DomainManager

dm = DomainManager()
allowed = dm.check_permission(
    agent_id="agent_001",
    domain="private:agent_001",
    action="read"
)
```

### 注册Agent

```python
dm.register_agent(
    agent_id="new_agent",
    role="normal",
    domains=["private:new_agent"]
)
```

### 授权共享域

```python
dm._grant(
    agent_id="agent_002",
    domain="shared:team_alpha",
    permissions=["read", "write"]
)
```

## 安全原则

1. **最小权限原则**：默认只赋予必要的最低权限
2. **默认拒绝**：未明确授权的操作一律拒绝
3. **审计追踪**：所有权限检查结果均有审计记录
4. **权限分离**：读权限和写权限分离管理
5. **定期审查**：共享域权限需定期审查和清理
