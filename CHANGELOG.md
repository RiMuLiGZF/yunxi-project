## [1.1.0] - 2026-07-18

### 发布概述

V1.1.0 是 V1.0 发布后的第一轮完整优化迭代版本。基于 V1.0 第一轮全面大检查发现的 P1 级问题，

### 安全加固

#### 🔴 高危问题修复

- **SEC-004: M8 控制塔 JWT 统一认证迁移** (CVSS 7.5)
  - M8 独立 JWT 实现迁移到 shared 统一 `JWTHandler`

  - 新增密钥轮换接口，支持运行时密钥轮换

- **SEC-005: M2 技能集群 SQL 注入防护** (CVSS 7.5)
  - 新增 `validate_identifier()` 等 4 个输入验证函数
  - `order_by`、`update_fields`、`like_search` 等动态 SQL 全部加白名单校验
  - 覆盖所有排序列（id、name、status、created_at、updated_at 等）
  - market/registry 动态 SQL 同步加固

- **SEC-008: 测试环境配置泄露加固** (CVSS 7.2)
  - 测试弱密码替换为随机强密码
  - 测试 JWT 密钥替换为 64 字符随机强密钥

## [0.8.0] - 2026-07-15

### 发布概述

v0.8.0 是分布式架构版本，代号"分布式"。本版本实现节点注册发现、
跨节点消息总线和联邦调度能力，将系统从单体架构升级为可水平扩展的分布式架构。

### 新增功能

- **节点注册发现**：基于服务注册表的节点自动发现和健康检查机制
- **跨节点消息总线**：M1 消息总线支持跨节点通信，实现分布式事件驱动
- **联邦调度**：M1 调度中心支持联邦调度，任务可在多节点间分配
- **服务发现与负载均衡**：支持轮询、加权轮询、最少连接等多种策略
- **故障转移机制**：主备模式和自动故障切换

### 改进优化

- **架构解耦**：模块间通过标准接口通信，降低耦合度
- **通信协议标准化**：统一 A2A 协议和 MCP 协议，减少适配成本
- **配置中心化**：支持集中式配置管理和动态配置更新
- **监控体系完善**：跨节点指标收集和统一监控面板
- **网络通信优化**：连接池和批量消息传输，降低网络开销

### 已知问题

1. 分布式一致性采用最终一致性模型，强一致场景需额外处理
2. 网络分区恢复后的状态同步可能存在延迟
3. 节点数量超过 10 台时注册表性能下降明显
4. 联邦调度的任务亲和性策略较为简单

## [1.0.1] - 2026-07-18

### 发布概述

V1.0.1 是 V1.0 发布后的第一个紧急安全修复版本。基于 V1.0 第一轮全面大检查发现的问题，修复了 6 个严重/高危安全问题和 2 个测试质量问题。建议所有生产环境用户立即升级。

### 安全修复

#### 🔴 严重问题修复

- **SEC-001: 全局模块默认 Token 硬编码** (CVSS 9.8)
  - 将 14 个模块的默认 admin Token 从可预测值改为空字符串
  - 生产环境强制校验 Token 必须通过环境变量显式配置
  - 开发环境自动生成随机 Token，日志一次性显示

- **SEC-002: 全局 JWT 默认密钥硬编码** (CVSS 9.8)
  - JWT 密钥默认值从 `yunxi-jwt-secret-key-2026` 改为空字符串
  - 生产环境强制校验：空密钥报错 + HS256 算法长度 ≥ 32 字符
  - 开发环境自动生成随机密钥
  - 弱密钥模式检测（`yunxi-` 前缀等）

- **SEC-003: 终端命令技能命令注入漏洞** (CVSS 9.8)
  - 彻底移除 `shell=True` 选项，所有命令以列表形式传递
  - 七层防御体系：命令解析 → 黑名单 → 白名单 → 参数检查 → 危险组合检测 → 工作目录限制 → 超时限制
  - 40+ 危险命令黑名单，60+ 常用命令白名单
  - 所有命令执行记录审计日志

- **ARC-001: M7 工作流 eval/exec 沙箱风险**
  - 已识别风险，将在 V1.1 中引入 AST 白名单解析方案
  - 临时缓解：确保工作流功能仅授权用户可使用

#### 🟠 高危问题修复

- **SEC-006: API 网关 JWT 开发模式不验证签名** (CVSS 7.5)
  - 彻底移除降级模式，jose 库不可用时直接阻止启动
  - 签名验证始终开启，无任何环境例外

- **SEC-007: 密码哈希开发模式降级为 SHA256** (CVSS 7.5)
  - 移除 `YUNXI_DEV_MODE=1` 自动降级逻辑
  - 新增 `YUNXI_INSECURE_PASSWORD=1` 显式开关（需用户主动确认）
  - Fallback 模式启用时打印醒目红色警告

- **SEC-012: 文件操作技能路径遍历加固** (CVSS 5.5)
  - 使用 `os.path.realpath()` 解析符号链接后校验
  - 七层路径安全检查
  - 符号链接检测与日志记录
  - 文件大小限制（读取 10MB，写入 50MB）

### 测试质量改进

- **QAL-001: M8 重复测试文件清理**
  - 删除 M8 backend 根目录下 8 个重复测试文件
  - 统一收敛到 `backend/tests/` 目录

- **TST-001: pytest.skip 模式清理（第一阶段）**
  - 5 个重点文件修复，skip 调用减少 41 处（-18.2%）
  - 纯单元测试标记 `@pytest.mark.unit`，移除 try/except+skip
  - 需要真实服务的测试标记 `@pytest.mark.integration`
  - pytest.ini 默认排除 integration/e2e/performance/slow 标记

### 新增测试

- 生产环境：缺少 Token/密钥配置时将阻止启动（安全强化，可能需要补充配置）

## [1.0.0] - 2026-07-17

### 发布概述

云汐系统 V1.0 正式发布，这是一个里程碑版本，标志着系统从开发阶段进入生产就绪阶段。
本版本包含完整的高可用架构、数据容灾体系、故障演练框架，以及全功能的模块系统。

### 新增功能

#### 高可用架构 (HA)
- **服务发现与健康检查增强**
  - 基于模块注册表的服务发现机制
  - 多维度健康检查：HTTP、TCP 端口、依赖服务、系统资源（CPU/内存/磁盘）
  - 不健康实例自动摘除与自动恢复机制
  - 健康检查历史记录与趋势分析

- **负载均衡**
  - 轮询 (Round Robin)
  - 加权轮询 (Weighted Round Robin) - 平滑加权算法
  - 最少连接 (Least Connections)
  - 最快响应 (Fastest Response)
  - 一致性哈希 (Consistent Hash) - 支持虚拟节点

- **故障转移**
  - 主备模式 (Active-Passive)
  - 故障自动检测与切换
  - 切换前状态同步
  - 自动恢复检查与回切
  - 切换冷却时间，防止抖动

- **服务注册表**
  - 实例注册与注销
  - 心跳保活机制
  - 健康实例自动发现
  - 实例元数据管理

#### 数据容灾 (DR)
- **备份增强**
  - 全量备份：完整数据库备份（SQLite 在线备份 API）
  - 增量备份：基于 WAL 日志的增量备份
  - 定时备份：支持间隔式、每日定时、每周定时
  - 备份验证：四级验证（快速/校验和/完整性/完全）
  - 异地备份：支持备份到远程存储（本地目录模拟）
  - 备份保留策略：按数量自动清理

- **数据恢复**
  - 全量恢复：从全量备份恢复
  - 时间点恢复 (PITR)：基于 WAL 日志重放
  - 增量恢复：全量 + 增量链恢复
  - 恢复前自动备份当前数据（安全网机制）
  - 恢复验证：完整性检查、表数量、行数估算
  - 恢复进度实时跟踪
  - 失败自动回滚

- **数据库高可用**
  - SQLite WAL 模式优化配置
  - 连接池管理：连接复用、并发限制、健康检查
  - 读写分离设计（概念层接口）
  - 数据库故障检测与自动恢复
  - 实时健康监控

#### 故障演练框架 (Chaos Engineering)
- **故障注入工具**
  - 模块故障模拟（宕机模拟）
  - 网络延迟注入（可配置延迟和抖动）
  - 错误响应注入（可配置错误率和状态码）
  - 资源耗尽模拟（CPU、内存）
  - 磁盘 IO 变慢模拟
  - 连接断开模拟
  - 所有故障支持自动恢复

- **预设演练脚本**
  - 模块宕机恢复演练
  - 数据库故障切换演练
  - 网络分区演练
  - 全系统故障恢复演练

- **演练报告**
  - 演练目标和范围说明
  - 故障注入步骤详细记录
  - 系统响应事件时间线
  - 恢复时间统计（MTTR、RTO/RPO 评估）
  - 问题发现与改进项
  - 多格式导出：JSON、Markdown、HTML

### 已知问题和限制

1. **SQLite 限制**：SQLite 作为主数据库，在超高并发写入场景下性能有限。
   建议读写比 > 9:1 的场景使用。如需更高写入性能，可迁移到 PostgreSQL。

2. **单机部署**：当前版本主要面向单机部署，分布式部署需要额外配置。
   高可用架构已预留接口，可扩展为多节点部署。

3. **增量备份限制**：SQLite 的增量备份基于 WAL 日志，
   如果数据库执行了检查点，部分增量数据可能已合并到主文件。

4. **演练框架**：故障注入为模拟实现，实际生产环境的故障注入需要
   结合操作系统和网络层工具。

5. **异地备份**：当前异地备份使用本地目录模拟，
   生产环境建议配置到真实的远程存储（如对象存储、远程服务器）。

### 升级指南

#### 从开发版升级到 V1.0

1. **备份数据**
   ```bash
   # 备份所有数据库
   python -c "from data_layer import BackupManager; bm = BackupManager('./backups', './data'); bm.backup_database('m0.db')"
   ```

2. **更新代码**
   ```bash
   git pull origin main
   git checkout v1.0.0
   ```

3. **更新依赖**
   ```bash
   pip install -r requirements.txt
   ```

4. **执行数据库迁移**
   ```bash
   python -m migrations.upgrade
   ```

5. **更新配置**
   - 复制 `config/settings.prod.yaml` 为 `config/settings.yaml`
   - 根据实际环境修改配置
   - 配置备份策略和高可用参数

6. **验证安装**
   ```bash
   # 运行健康检查
   python scripts/health-check.ps1
   
   # 运行冒烟测试
   python -m pytest tests/smoke/ -v
   ```

7. **启动服务**
   ```bash
   # Windows
   scripts/start-all.ps1
   ```

#### 回滚方案

如升级后出现问题，可按以下步骤回滚：

1. 停止所有服务
2. 恢复数据库备份
3. 切换回旧版本代码：`git checkout <旧版本>`
4. 恢复旧版本配置
5. 重启服务
