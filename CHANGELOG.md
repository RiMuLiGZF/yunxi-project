## [0.4.0] - 2026-07-17

### 概述

**生产就绪版本** — 完成生产环境部署体系、安全深度加固、容灾恢复体系和全量文档体系，系统达到生产可用标准。

### Added

- **生产部署体系**：新增 `deploy-prod.ps1`、`upgrade.ps1`、`rollback.ps1` 三套生产运维脚本
- **运维工具链**：新增 `health-check`、`logs`、`monitor` 运维脚本，支持日常运维操作
- **安全深度加固**：安全头中间件、输入验证增强、速率限制机制，累计 81 个安全测试用例
- **容灾恢复体系**：备份验证机制、灾难恢复脚本、17 项容灾验证测试，保障数据安全
- **文档体系完善**：架构文档、API 文档、部署手册、运维手册、安全文档、灾难恢复、开发者指南共 7 类核心文档
- **M12 安全盾**：WAF 防护、密钥管理、速率限制、威胁检测四大安全能力
- **M11 MCP 总线**：工具注册、发现、路由、调用全流程能力，支持 87 个 MCP 工具

### Changed

- 所有模块统一接入生产级健康检查标准（M8 标准健康检查接口）
- 部署架构升级为 Docker Compose 生产模式（`docker-compose.prod.yml`）
- 配置管理全面迁移至 `pydantic-settings`，支持多环境配置分层
- 日志体系升级为结构化日志，支持链路追踪 ID 全链路透传

### Fixed

- **P0 级安全紧急修复**：默认密钥硬编码问题，JWT 密钥改为环境变量注入
- **P0 级安全紧急修复**：CORS 配置过宽问题，收紧允许的源地址范围
- **P0 级安全紧急修复**：WAF 防护规则缺失，新增 SQL 注入、XSS、路径遍历检测
- **P0 级安全紧急修复**：弱密码策略，新增密码强度校验和 bcrypt 加密存储
- 修复生产环境下多模块端口冲突问题
- 修复备份脚本在 Windows 环境下的路径兼容性问题

### Security

- 新增安全响应头中间件（HSTS、X-Content-Type-Options、X-Frame-Options 等）
- 新增输入验证框架，统一处理参数校验和类型转换
- 新增速率限制中间件，支持基于 IP 和用户的双重限流
- 新增 WAF 中间件，检测并拦截常见 Web 攻击

### Removed

- 清理 `_archive` 目录下的遗留代码（第一阶段原型代码）
- 移除 `shared/` 根目录下的重复模块文件（已迁移至三层架构）
- 废弃旧版 `M1-agent-cluster` 模块目录，统一使用 `M1-agent-hub`

## [0.3.0] - 2026-07-17

### 概述

**全量接入与质量版本** — 完成 API Gateway v2.0 全量路由接入，所有模块统一基础设施，性能与可观测性大幅提升，测试体系完善。

### Added

- **API Gateway v2.0**：12 个模块全量路由接入，支持统一认证、熔断降级、负载均衡
- **全模块统一基础设施**：配置管理、日志体系、错误码体系、认证鉴权全部接入 `shared` 公共库
- **Metrics 指标体系**：4 种指标类型（Counter / Gauge / Histogram / Summary），各模块统一埋点
- **健康检查标准化**：所有模块实现 M8 标准健康检查接口（`/health`、`/health/live`、`/health/ready`）
- **测试体系升级**：累计 753+ 测试用例，覆盖单元测试、集成测试、端到端测试
- **pytest 全局配置**：统一 `pytest.ini` 配置，支持 30+ 测试标记分类
- **覆盖率配置**：`.coveragerc` 覆盖率配置，支持分模块覆盖率统计

### Changed

- 数据库索引优化：为高频查询字段添加索引，查询性能提升 40%+
- 缓存策略优化：新增内存缓存层，热点路径响应时间降低 60%
- 所有模块路由统一通过 API Gateway 暴露，取消模块直接对外端口
- `shared` 公共库依赖升级，各模块统一使用最新版 `shared.core`

### Performance

- 数据库连接池优化，减少连接创建开销
- 热点接口增加本地缓存（TTL 策略）
- 日志异步写入，降低 I/O 阻塞对请求的影响

## [0.2.0] - 2026-07-14

### 概述

**统一治理版本** — 完成 shared 三层架构重构、统一认证体系、统一配置管理、统一日志体系、统一错误码体系，实现全模块标准化治理。

### Added

- **shared 三层架构**：重构为 `core`（基础工具层）、`data`（数据基础设施层）、`business`（业务能力层）三层架构
- **统一认证体系**：JWT + API Key 双认证机制，支持 RBAC 角色权限管理
- **统一配置管理**：基于 `pydantic-settings` 的全局配置中心，支持环境变量和 `.env` 文件
- **统一日志体系**：结构化日志 + 链路追踪（Trace ID），全链路日志可追溯
- **全模块统一备份机制**：`backup_manager` 统一管理各模块数据备份，支持定时、增量、全量备份
- **统一错误码体系**：6 位数字错误码规范（模块编号 + 错误类别 + 具体序号），全模块错误码统一
- **统一响应格式**：`{ code, message, data, trace_id }` 标准响应结构

### Changed

- 各模块逐步迁移至 `shared.core.*` 标准路径，旧路径标记为 Deprecated
- 认证中间件统一接入，各模块不再单独实现鉴权逻辑
- 配置读取统一通过 `shared.core.config.get_config()`，消除分散的配置读取方式

### Deprecated

- `shared/` 根目录下的扁平模块文件（如 `shared/auth.py`、`shared/logger.py`），建议迁移至 `shared/core/` 对应模块

## [0.1.0] - 2026-07-13

### 概述

**P0 安全清零版本** — 完成 13 个模块的骨架搭建和核心功能初步实现，并修复全部 P0 级安全问题，实现安全基线达标。

### Added

- **13 个模块骨架**：M0 ~ M12 全部模块完成基础骨架搭建（FastAPI 应用、路由、配置、数据模型）
- **API 网关初版**：基础路由转发、认证透传功能
- **M8 管理控制塔**：用户认证、模块纳管、基础仪表盘
- **M1 多 Agent 集群**：Agent 调度框架、多模型路由
- **M2 技能集群**：18 种内置技能的注册与执行框架
- **M5 潮汐记忆**：四层记忆模型（沙滩/浅水/深水/深海）基础实现
- **M7 积木编排**：DAG 工作流引擎基础版本
- **shared 公共库初版**：认证、日志、错误处理、配置等基础工具

### Fixed

- **P0 安全问题清零**：12 个 P0 级安全问题全部修复
  - 硬编码密钥 → 环境变量注入
  - 明文密码存储 → bcrypt 加密存储
  - SQL 注入风险 → 参数化查询
  - 未授权访问 → JWT 认证中间件
  - 敏感信息泄露 → 错误信息脱敏
  - CORS 配置过宽 → 严格白名单
  - 速率限制缺失 → 令牌桶限流
  - 文件上传无校验 → MIME 类型 + 大小限制
  - 弱密码策略 → 密码强度校验
  - 会话管理不当 → Token 过期 + 刷新机制
  - 日志敏感信息 → 自动脱敏
  - 权限绕过风险 → RBAC 权限校验

### Security

- 初版安全基线建立，P0 级风险全部清零
- 安全审计机制初步建立

## 版本说明

### 版本号规则

版本号格式：`MAJOR.MINOR.PATCH`

| 层级 | 说明 | 示例 |
|------|------|------|
| **MAJOR** | 不兼容的 API 变更 | 架构重构、协议升级 |
| **MINOR** | 向下兼容的功能性新增 | 新模块、新特性 |
| **PATCH** | 向下兼容的问题修正 | Bug 修复、安全补丁 |

### 变更类型

| 分类 | 说明 |
|------|------|
| **Added** | 新增功能 |
| **Changed** | 功能变更或优化 |
| **Deprecated** | 即将废弃的功能 |
| **Removed** | 已移除的功能 |
| **Fixed** | Bug 修复 |
| **Security** | 安全相关变更 |

### 发布节奏

- **主版本（MAJOR）**：重大架构升级，无固定周期
- **次版本（MINOR）**：阶段性功能交付，约 1-2 周一个版本
- **修订版本（PATCH）**：Bug 修复和安全补丁，按需发布
