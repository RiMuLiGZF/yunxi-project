## 五、修复与优化内容

### 5.1 P0-1 修复: MCP logger 崩溃

**修复方式**:
```python
# 在 McpClientBridge.__init__ 中添加
self._logger = logger.bind(skill_id=skill_id, mcp_server=mcp_server_url)
```

使用 structlog.bind() 绑定上下文，与现有代码风格一致。

### 5.2 P0-2 修复: 中间件接入主调用链

**修复方式**:
1. SkillRouter.__init__ 新增 middleware 参数
2. 新增 use() 方法支持链式注册
3. invoke() 方法通过中间件管道执行（洋葱模型）
4. 原有四阶段逻辑抽取为 _invoke_core() 方法

**执行顺序（洋葱模型）**:
```
请求 → 缓存 → 事件 → 弹性 → 指标 → 日志 → 核心逻辑（权限/解析/依赖/执行）→ 日志 → 指标 → 弹性 → 事件 → 缓存 → 响应
```

### 5.3 P0-3 修复: 新增统一钩子系统

**新增文件**: `skill_cluster/hooks.py`

**8个标准钩子点**:
- `before_invoke` - 调用前
- `after_invoke` - 调用后
- `on_error` - 错误时
- `before_register` - 技能注册前
- `after_register` - 技能注册后
- `before_unregister` - 技能卸载前
- `system_startup` - 系统启动
- `system_shutdown` - 系统关闭

**核心能力**:
- 优先级支持（数字越小越先执行，默认100）
- 短路返回（钩子返回非None即终止后续）
- 三种注册方式: register() / @hooks.on() 装饰器 / 事件总线桥接
- 与中间件打通: create_middleware() 生成可直接注册的中间件
- 与事件总线打通: subscribe_event() 桥接事件总线

**测试覆盖**: 21 个单元测试，全部通过

### 5.4 额外优化

1. **MCP桥接接入中间件**: McpClientBridge 也支持中间件管道，MCP调用可享受缓存/熔断/指标
2. **MCP initialize 握手**: 补充标准 initialize/initialized 协议流程
3. **开发期进程监控**: 全程监控系统进程，确保不启动多余进程
