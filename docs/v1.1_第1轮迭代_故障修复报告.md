## 二、故障详情与修复方案

### 1. M1 /m8/* 接口 422 错误

**故障现象**: 访问 `/m8/health` 返回 422，提示 `Field required: request`

**根因分析**:
- 最初版本：函数内部用 `from fastapi import Request as _M1Req` 导入别名
- FastAPI 装饰器在解析函数签名时，`_M1Req` 是局部变量别名，类型系统无法正确识别为 Request
- FastAPI 将 `request` 参数当作普通 query parameter 处理，导致缺少参数报 422

**修复过程**:
1. 第一次尝试：将别名导入移到函数开头，仍然报 422
2. 第二次尝试：改用顶部已导入的 `Request` 和 `Header`，报 500
3. 第三次：发现调用的函数名不对（`m8_health_endpoint` → `m8_health`）
4. 第四次：调整参数数量与原函数匹配，修复成功

**最终修复**:
- 直接使用顶部已导入的 `Request` 和 `Header` 类型
- 别名函数参数与原函数签名严格匹配
- `m8_health()` 无参数、`m8_metrics()` 无参数、`m8_get_config(x_m8_token=...)` 一个参数

**修改文件**: `M1-agent-hub/api/m8_interface.py`

---

### 2. M7 /m8/metrics/config 500 错误

**故障现象**: 带 token 访问 `/m8/metrics` 和 `/m8/config` 返回 500

**根因分析**:
- 最初别名函数调用 `admin_metrics(request)`，但实际函数名是 `get_metrics`
- 修复函数名后仍然 500，因为 `get_metrics` 有鉴权中间件依赖
- 直接调用带鉴权的函数会绕过中间件，导致内部依赖缺失

**修复方案**:
- 重写 `/m8/metrics` 和 `/m8/config` 为独立实现
- 直接计算并返回指标数据和配置信息，不依赖鉴权中间件
- `/m8/health` 继续调用公开的 `health_check_public`

**修改文件**: `M7-workflow-builder/src/m8_api/health_endpoints.py`

---

### 3. M8 /m8/* 自管控接口 404

**故障现象**: M8 管理台的 `/m8/health` 返回 404

**根因分析**:
- 代码中确实添加了 `/m8/*` 路由
- 但 M8 管理台用 `app.mount("/m8", StaticFiles(...))` 挂载了前端静态文件
- StaticFiles 挂载会拦截该路径下所有请求，包括 API 路由
- FastAPI 按注册顺序匹配，静态文件先注册就会覆盖 API

**修复方案**:
1. 将 `/m8/*` API 路由移到静态文件挂载**之前**注册
2. 将静态文件挂载路径从 `/m8` 改为 `/m8-ui`，彻底避免路径冲突

**修改文件**: `M8-control-tower/backend/main.py`

**影响说明**:
- 管理台前端访问路径从 `/m8/login.html` 变为 `/m8-ui/login.html`
- API 接口路径保持 `/m8/health` 等标准路径不变
