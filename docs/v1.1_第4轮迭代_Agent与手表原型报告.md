## 二、UI-05: Agent 中心界面交互重构

### 布局升级（三栏布局）
- **左侧分类侧边栏**（200px）：我的 Agent、自定义分组、其他分类，每项带数量统计
- **中间卡片区域**：工具栏 + 批量操作栏 + 卡片网格/列表
- **右侧详情面板**（420px）：点击卡片从右侧滑出，带遮罩层

### 核心功能

#### 1. 卡片展示
- Agent 头像（服务商色+首字母）+ 状态徽标（在线/忙碌/离线，带脉冲动画）
- 名称、服务商、描述（两行截断）
- 能力标签（最多3个）
- 调用次数 + 成功率统计
- 底部快捷操作：启动/停止、配置、删除
- 悬停效果：上浮 4px + 阴影增强 + 边框高亮

#### 2. 拖拽功能（原生 HTML5 Drag and Drop API）
- 卡片间拖拽排序（视觉反馈：半透明+旋转+目标高亮）
- 拖拽到左侧分组：移动 Agent 到目标分组
- 拖拽过程中源卡片半透明倾斜，目标区域高亮边框

#### 3. 批量操作
- 点击「批量操作」按钮进入多选模式
- 全选 / 反选功能
- 批量启动、批量停止、移动分组、导出配置、批量删除
- 顶部粘性批量操作栏显示已选数量

#### 4. 搜索与筛选
- 顶部搜索框：名称/描述模糊搜索（实时过滤）
- 标签筛选：对话/代码/图像/语音/推理
- 状态筛选：全部/在线/忙碌/离线
- 排序：名称/调用次数/创建时间 升降序
- 网格视图 / 列表视图切换

#### 5. 详情面板（右侧滑出）
- Agent 基本信息（大头像 + 状态 + 类型标签）
- 操作按钮组（启动/停止 + 配置 + 收藏）
- 统计卡片（总调用次数 + 成功率）
- 能力/技能标签列表
- 调用趋势折线图（ECharts 渐变面积图，15天数据）
- 配置信息（模型、价格、温度、最大Token、创建时间）
- 最近调用记录列表

#### 6. 交互细节
- 骨架屏加载（4个占位卡片，800ms 后显示真实数据）
- 空状态提示（图标 + 文字）
- Toast 提示（成功/错误/警告/信息四种类型）
- 确认删除模态框
- ESC 键关闭详情面板和弹窗
- 平滑过渡动画（150-300ms）
- 深色/浅色主题切换（ECharts 图表同步切换）

### 模拟数据
12 个 Agent（涵盖对话、图像、语音、嵌入、代码类型，9个服务商，含在线/忙碌/离线三种状态）

## 三、WATCH-04: 手表端极简 UI 原型

### 文件
`frontend/watch/index.html`

### 设计特点
- **圆形屏幕模拟**: 390x390px 圆形屏幕，border-radius: 50% 实现
- **深色OLED主题**: 纯黑背景 + 高对比度文字，适合OLED屏幕省电
- **CSS变量主题系统**: 统一的颜色、间距、动画变量
- **大字体大点击区域**: 适合手指在小屏幕上操作

### 五个页面视图
1. **表盘主页** - 实时时钟/日期 + 心率/步数/电量三个SVG状态环（带动画）
2. **健康数据页** - 心率/血氧/步数/卡路里 四张数据卡片（2x2网格）
3. **通知中心页** - 可滚动通知列表，含微信/云汐助手/健康/日程等模拟通知
4. **快捷操作页** - 语音备忘/快速记录/提醒设置三个大按钮
5. **云汐助手页** - 极简对话界面，支持文字输入和语音按钮

### 交互功能
- **触摸滑动**: 支持左右滑动切换页面，带边界阻尼效果
- **鼠标拖拽**: 桌面端可用鼠标拖拽模拟滑动
- **键盘导航**: 左右方向键切换页面
- **指示器点击**: 底部5个圆点指示器可点击跳转
- **实时时钟**: 秒级更新的数字时钟
- **动态数据**: 心率/步数每5秒模拟波动
- **对话交互**: 云汐助手支持输入消息并模拟回复
- **语音模拟**: 录音按钮有脉冲动画，模拟语音转文字

## 四、TEST-06: 测试报告自动生成与归档

### 工具文件
- `scripts/test/run_tests.py` - 测试运行入口脚本
- `scripts/test/generate_report.py` - 报告生成器核心模块
- `docs/test-reports/` - 测试报告归档目录

### 核心功能

#### run_tests.py
- 封装 pytest 运行，自动生成 JSON 结果
- 测试完成后自动调用报告生成器
- 自动归档到 `docs/test-reports/` 目录
- 自动生成 `latest.html` 和 `latest.md` 快捷入口
- 支持按路径、marker、关键字过滤测试
- 支持安静模式、仅报告模式等多种选项

#### generate_report.py
**TestReportGenerator 核心类**：
- 读取 pytest-json-report 输出的 JSON 结果
- 自动统计：总用例数、通过数、失败数、跳过数、错误数、通过率、耗时
- 按模块分类统计（从测试路径提取模块名）
- 失败用例详情展示（错误信息 + 完整堆栈）
- 运行环境信息（Python版本、系统、平台、架构、时间）

### 输出格式
- **HTML报告**: 深色主题、响应式设计、CSS饼图、进度条、可折叠失败详情、统计卡片
- **Markdown报告**: 结构化表格，适合归档到docs目录

### 使用示例
```bash
# 运行全部测试并生成报告
python scripts/test/run_tests.py

# 运行冒烟测试
python scripts/test/run_tests.py -m smoke

# 运行指定模块
python scripts/test/run_tests.py tests/test_m8/

# 按关键字过滤
python scripts/test/run_tests.py -k "auth and not slow"
```
