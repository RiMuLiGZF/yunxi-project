## 二、UI-01: 仪表盘数据可视化

### 实现内容

#### 1. 顶部状态卡片（4个）
- **在线模块数**：环形进度指示，显示 8/8 运行中
- **今日任务数**：含同比变化标签
- **算力调用次数**：今日累计，带 CPU 图标
- **系统健康度评分**：百分制，优秀/良好提示

#### 2. 图表区域（ECharts 实现）
- **模块运行状态柱状图**：双 Y 轴展示运行时长和请求数，渐变色柱体
- **算力调用趋势折线图**：近 7 天调用次数 + 成功率双折线，面积填充
- **任务完成率饼图**：环形饼图，中心显示完成率百分比，4 种状态图例

#### 3. 快捷操作区（8个入口）
- 启动模块、停止模块、一键系统检查、算力调度
- Agent 管理、系统监控、部署中心、系统设置
- 悬停效果：背景色变化 + 边框高亮 + 上浮动画 + 阴影

#### 4. 最近动态列表
- 时间线式展示系统事件
- 按时间倒序排列，带状态图标和来源标识

### 技术特点
- 纯 HTML + JS + ECharts 5.5（CDN 引入）
- 深色主题风格，与 compute.html 设计语言一致
- 响应式布局（grid 网格）
- 数据先用模拟数据，API 接口已预留对接位置

### 修改文件
- `frontend/m8/dashboard.html` - 全面升级数据可视化

## 三、TEST-01: 自动化测试框架搭建

### 目录结构
```
tests/
├── conftest.py          # 全局 fixtures
├── pytest.ini           # （项目根目录）
├── .env.test            # （项目根目录）测试环境变量
├── utils/
│   ├── api_client.py    # 统一 API 测试客户端
│   └── data_generator.py # 测试数据生成器
├── test_m8/             # M8 管理台测试
│   ├── test_health_check.py
│   ├── test_auth_api.py
│   └── test_compute_api.py
├── test_m1/             # M1 调度模块测试
│   ├── test_startup_check_agent.py
│   └── test_m8_integration.py
├── test_integration/    # 集成测试
│   └── test_dashboard_flow.py
└── reports/             # 测试报告输出目录
```

### 核心能力
1. **统一 API 客户端**：自动处理认证、响应解析、超时、错误处理、响应时间统计
2. **测试数据生成器**：用户、任务、模块状态、算力调用、系统统计等数据生成
3. **全局 fixtures**：M8 应用实例、TestClient、admin token、测试数据库、断言辅助工具
4. **测试标记体系**：smoke/integration/unit/slow/m8/m1/compute/health/auth 等

### 测试覆盖（46 个用例）
| 模块 | 用例数 | 说明 |
|------|--------|------|
| M8 健康检查 | 8 | 健康检查、系统检测、模块列表等 |
| M8 认证 API | 9 | 登录、token验证、权限校验等 |
| 算力调度 API | 8 | 算力源CRUD、分组、模型、路由等 |
| 启动检查 Agent | 12 | 6大检查项验证、结果存储、历史查询 |
| M1-M8 集成 | 8 | 模块间通信、接口调用等 |
| 仪表盘数据流 | 6 | 数据格式、统计计算等 |
