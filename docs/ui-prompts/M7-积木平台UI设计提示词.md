# M7 积木编排平台 - UI 界面设计提示词

> 本文档用于交给前端/UI 设计对话框，生成 M7 积木编排平台的前端界面。
> 技术栈建议：Vue 3 + Vite + Element Plus / React + Vite + Ant Design
> 核心功能：可视化拖拽式工作流编排，像搭积木一样构建 AI 工作流
> 设计风格：简洁现代、浅色主题、拖拽交互、节点化编排

---

## 一、项目概述

**产品名称**：云汐积木编排平台（M7 Blocks Platform）
**产品定位**：低代码可视化工作流编排平台，通过拖拽积木式节点构建 AI 工作流
**目标用户**：产品经理、运营人员、开发者
**核心价值**：无需编码，通过可视化拖拽快速构建复杂的 AI 业务流程

## 三、页面清单

### 页面 1：首页 / 工作流列表 `/`

**功能**：展示所有工作流项目，支持创建和管理

**设计要点**：

**顶部**：
- 左侧：Logo + "积木编排平台" 标题
- 右侧：搜索框、「新建工作流」按钮（主按钮）、模板库入口

**工作流列表**（卡片网格）：
- 每个卡片：封面缩略图、工作流名称、描述、更新时间、状态（草稿/已发布）
- 卡片操作：编辑、运行、复制、删除
- 悬浮效果：卡片上浮 + 阴影加深

**分类筛选**：
- 全部 / 我的 / 最近编辑 / 已发布 / 模板

---

### 页面 2：工作流编辑器 `/workflow/:id`

**功能**：核心编辑页面，可视化拖拽编排工作流

**设计要点**：

#### 左侧：组件库面板

**可折叠分类**：
- 🟢 **触发块**（开始节点）
  - 手动触发、定时触发、Webhook 触发、API 触发
- 🟣 **AI 块**（大模型相关）
  - 大模型对话、文本生成、代码生成、图像生成、语音合成
- 🟡 **逻辑块**（流程控制）
  - 条件判断、分支选择、循环、等待、并行执行
- 🔵 **数据块**（数据处理）
  - 变量设置、数据转换、JSON 解析、HTTP 请求、数据库查询
- 🟠 **工具块**（外部能力）
  - 邮件发送、文件处理、日历、通知推送、计算器
- ⚙️ **自定义块**（用户自定义）

**组件拖拽**：
- 从左侧拖拽到画布即可创建节点
- 拖拽时显示半透明预览
- 支持搜索过滤组件

#### 中部：画布区域

**画布功能**：
- 无限画布，可平移（拖拽空白处）
- 可缩放（鼠标滚轮，底部有缩放控制）
- 节点可拖拽移动
- 节点之间连线（从输出端口拖到输入端口）
- 框选多个节点，整体移动
- 右键菜单：复制、删除、对齐、编组

**节点样式**：
- 顶部彩色条（按类型着色）+ 图标 + 节点名称
- 左侧输入端口（小圆点），右侧输出端口（小圆点）
- 选中时：边框加粗 + 阴影
- 运行中：节点边框呼吸动画 + 状态指示器

**连线样式**：
- 贝塞尔曲线连接
- 连线中间可加标签（条件分支时显示条件）
- 选中连线可删除

#### 右侧：属性面板

**无选中时**：
- 显示工作流全局设置
  - 工作流名称、描述
  - 版本号
  - 全局变量
  - 运行配置（超时、重试）

**选中节点时**：
- 节点基本信息（名称、类型、ID）
- 节点配置表单（根据节点类型动态变化）
- 输入配置：输入字段映射（支持变量引用）
- 输出配置：输出字段说明
- 错误处理：失败时策略（重试/跳过/终止）
- 调试：测试运行此节点

#### 顶部工具栏

- 保存（自动保存提示）
- 运行（▶ 按钮，点击后切换到调试模式）
- 发布（版本发布）
- 撤销 / 重做
- 缩放控制（适应画布 / 100% / +/-）
- 设置（工作流设置）
- 更多（导出JSON、复制链接、删除）

#### 底部状态栏

- 缩放比例（可点击调整）
- 节点数量
- 连线数量
- 保存状态（已保存 / 保存中...）
- 调试控制台开关（点击展开底部日志面板）

### 页面 3：运行调试页 `/workflow/:id/run`

**功能**：运行工作流并查看执行结果

**设计要点**：

**布局**：
- 左侧：工作流画布（只读，显示执行状态）

**运行状态可视化**：
- 节点按执行顺序依次高亮
- 已完成：绿色勾选标记

- Tab 切换：执行日志 / 输出结果 / 变量面板

**操作按钮**：
- 重新运行
- 停止运行
- 查看详情
- 保存为模板

## 四、核心交互规范

### 4.1 节点拖拽

1. 从左侧组件库拖拽组件到画布 → 创建节点
2. 画布内拖拽节点 → 移动位置
3. 拖拽节点边缘 → 调整大小（可选）
4. Shift + 点击 → 多选
5. 框选 → 批量选中

### 4.2 连线交互

1. 鼠标悬停在输出端口上 → 端口高亮放大
2. 按住从输出端口拖出 → 生成连线
3. 拖到输入端口上 → 端口高亮，松开完成连接
4. 点击连线 → 选中，按 Delete 删除
5. 同一输出可连多个输入（发散）
6. 同一输入只能连一个输出（汇聚需用特殊节点）

### 4.3 运行反馈

1. 点击运行 → 工作流从触发节点开始执行
2. 每个节点执行时有状态动画
3. 连线有流动光效（从左到右）
4. 执行完成后弹出结果摘要
5. 失败时高亮失败节点，显示错误信息

## 五、API 对接说明

后端 API 基地址：`http://localhost:3001/api/v1`

### 工作流接口
- `GET /workflows` — 获取工作流列表
- `POST /workflows` — 创建工作流
- `GET /workflows/{id}` — 获取工作流详情
- `PUT /workflows/{id}` — 更新工作流
- `DELETE /workflows/{id}` — 删除工作流

### 运行接口
- `POST /workflows/{id}/run` — 运行工作流
- `GET /runs/{run_id}` — 获取运行状态
- `GET /runs/{run_id}/logs` — 获取运行日志
- `POST /runs/{run_id}/stop` — 停止运行

### 积木块接口
- `GET /blocks` — 获取所有可用积木块
- `GET /blocks/{id}` — 获取积木块详情
- `POST /blocks` — 创建自定义积木块
- `PUT /blocks/{id}` — 更新自定义积木块

### 模板接口
- `GET /templates` — 获取模板列表
- `GET /templates/{id}` — 获取模板详情
- `POST /templates/{id}/use` — 使用模板创建工作流
