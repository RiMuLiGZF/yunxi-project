# M8 云汐管理台 - UI 界面设计提示词

> 本文档用于交给前端/UI 设计对话框，生成 M8 云汐管理台的前端界面。
> 技术栈建议：Vue 3 + Vite + Element Plus / React + Vite + Ant Design
> 设计风格：科技感、深色主题、数据可视化、模块化布局

---

## 一、项目概述

**产品名称**：云汐系统管理工作台（M8 Control Tower）
**产品定位**：云汐系统的统一管控中心，是 8 大模块的指挥室和监控大屏
**目标用户**：系统管理员、运维人员、产品运营
**核心价值**：一站式管理、监控、调度整个云汐系统

## 三、页面清单

### 页面 1：登录页 `/login`

**功能**：用户登录系统

**设计要点**：
- 居中卡片式布局，左侧品牌展示，右侧登录表单
- 背景：深色渐变 + 科技感粒子/网格动效
- Logo：云汐系统标志 + 副标题"智慧管控平台"
- 表单：用户名、密码、记住我、登录按钮
- 底部：版本号 v1.0.0

---

### 页面 2：总览仪表盘 `/dashboard`

**功能**：系统全局总览，一屏掌握系统状态

**设计要点**：

**顶部 4 个核心指标卡片**：
- 在线模块数：8 / 8（绿色圆环进度）
- 今日任务数：128（较昨日 +12%）
- 活跃用户数：36
- 系统运行时长：12天 6小时

**中部左侧：模块运行状态图**
- 8 个模块卡片列表，每个显示：模块名称、图标、状态灯（绿/黄/红）、CPU/内存占用
- 点击可跳转到模块详情

**中部右侧：任务趋势图**
- 折线图：近 7 天任务处理量趋势
- 柱状图：各模块任务分布

**底部：最近任务列表**
- 表格：任务ID、标题、模块、状态、耗时、开始时间
- 状态标签：completed（绿）、running（蓝）、queued（灰）、failed（红）

---

### 页面 3：任务中心 `/tasks`

**功能**：任务提交、查看、管理

**设计要点**：

**顶部操作栏**：
- 左侧：筛选标签（全部 / 运行中 / 已完成 / 失败 / 排队中）
- 右侧：「新建任务」按钮（主按钮）

**任务列表**：
- 卡片式列表或表格
- 每行：任务标题、模块标签、状态标签、进度条、耗时、创建时间、操作按钮
- 点击展开详情：输入内容、输出结果、执行步骤时间线

**新建任务弹窗**：
- 任务标题（输入框）
- 任务描述（文本域）
- 目标模块（下拉选择：M1调度 / M2技能 / M4场景 / M7积木）
- 优先级（单选：高 / 中 / 低）
- 提交按钮

---

### 页面 4：模块管理 `/modules`

**功能**：查看和管理 8 大模块

**设计要点**：

**模块卡片网格**（2行×4列）：
- 每个卡片：模块图标、名称、版本号、状态灯、端口号
- 鼠标悬停显示操作按钮：启动 / 停止 / 重启 / 配置
- 点击卡片进入模块详情页

**模块详情页**：
- 基本信息：名称、版本、端口、状态、运行时长
- 资源使用：CPU、内存、磁盘（仪表盘）
- API 接口列表：路径、方法、描述、调用次数
- 配置项：可编辑的配置表单（键值对）
- 操作按钮：启动、停止、重启、查看日志

---

### 页面 5：部署管理 `/deploy`

**功能**：模块部署、版本管理

**设计要点**：

**部署流水线图**：
- 可视化展示各模块的部署状态
- 状态：待部署 / 部署中 / 已部署 / 部署失败
- 操作：一键部署全部 / 单独部署

**版本管理**：
- 表格：版本号、发布时间、发布说明、操作（回滚）
- 当前版本高亮标记

---

### 页面 6：系统监控 `/monitor`

**功能**：实时监控系统运行状态

**设计要点**：

**实时指标看板**：
- 系统 CPU / 内存 / 磁盘 / 网络（四个仪表盘）
- 实时刷新（3秒间隔）

**模块健康度雷达图**：
- 8 个维度（8 个模块）的健康度雷达图
- 分值 0-100

**告警列表**：
- 最新告警列表，按时间倒序
- 级别：严重 / 警告 / 提示
- 支持标记已读、批量处理

---

### 页面 7：系统设置 `/settings`

**功能**：系统配置、用户管理

**设计要点**：

**左侧 Tab 切换**：
- 基本设置
- 用户管理
- 权限角色
- API 密钥管理
- 安全设置

**基本设置**：
- 系统名称、Logo、主题切换（深色/浅色）
- 大模型配置（Provider、API Key、模型选择）
- 全局参数配置

**用户管理**：
- 用户列表表格（用户名、角色、状态、创建时间）
- 新增/编辑用户弹窗
- 重置密码操作

## 四、交互规范

1. **加载状态**：所有异步操作显示骨架屏或 Loading 动画
2. **操作反馈**：成功/失败使用 Toast 消息提示（3秒自动消失）
3. **危险操作**：删除、停止等危险操作需要二次确认弹窗
4. **空状态**：无数据时显示空状态插画 + 引导文字
5. **响应式**：支持 1280px 以上分辨率，平板自适应
6. **快捷键**：Ctrl+K 快速搜索，Esc 关闭弹窗

## 五、API 对接说明

后端 API 基地址：`http://localhost:8080/api`

### 认证接口
- `POST /auth/login` — 登录
- `POST /auth/logout` — 登出
- `GET /auth/me` — 获取当前用户

### 任务接口
- `POST /tasks/submit` — 提交任务
- `GET /tasks/` — 任务列表
- `GET /tasks/{task_id}` — 任务详情
- `POST /tasks/{task_id}/cancel` — 取消任务

### 模块接口
- `GET /modules/` — 模块列表
- `GET /modules/{module_id}` — 模块详情
- `GET /modules/{module_id}/health` — 模块健康检查

### 系统接口
- `GET /system/health` — 系统健康状态
- `GET /system/stats` — 系统统计数据

> 所有接口（除登录外）需要在 Header 中携带 `Authorization: Bearer <token>`
