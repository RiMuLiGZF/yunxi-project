## 三、定制修改方案

### 优化 1：统一错误响应格式

**方案**：
- 所有模块统一使用 `{code, message, data, request_id, timestamp}` 格式
- code=0 表示成功，非 0 表示错误
- HTTP 状态码与业务 code 分离

### 优化 2：规范日志输出

**方案**：
- 统一使用 structlog 作为日志库
- 日志格式：`[时间] [级别] [模块] 消息 - 关键参数`
- 关键操作必须记录请求 ID

### 优化 3：请求 ID 追踪

**方案**：
- 所有 HTTP 请求注入 X-Request-Id
- 跨模块调用时透传请求 ID
- 日志中包含请求 ID 便于链路追踪
