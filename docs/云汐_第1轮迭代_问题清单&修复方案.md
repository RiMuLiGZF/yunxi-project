## 三、定制修改方案

### 修复 1：M5 记忆存储接口

**根因**：M5 使用 `archive` 作为写入接口名，与预期的 `store` 不一致。

**修改文件**：`M5-tide-memory/server.py`

**修改内容**：
- 新增 `/api/v1/memory/store` POST 端点
- 作为 `archive` 的别名，调用相同的 `api_router.archive()` 方法
- 保持原有 `archive` 接口不变，确保向后兼容

**验证方式**：POST `/api/v1/memory/store` 返回 code=0

### 修复 2：M6 设备类型接口

**根因**：M6 未实现设备类型列表端点。

**修改文件**：`M6-hardware-peripheral/m6_hardware/api/devices.py`

**修改内容**：
- 新增 `/api/v1/devices/types` GET 端点
- 返回 6 种设备类型（watch/ring/desktop/ar/drone/laptop）
- 每种类型包含 key、name、description 字段

**验证方式**：GET `/api/v1/devices/types` 返回 6 种设备类型

### 修复 3：M8 Dashboard 侧边栏导航

**根因**：dashboard.html 侧边栏未包含 Agent 中心和算力调度入口。

**修改文件**：`frontend/m8/dashboard.html`

**修改内容**：
- 在「系统监控」和「系统设置」之间添加「Agent 中心」导航项
- 添加「算力调度」导航项
- 使用 Lucide 图标：`bot`（Agent 中心）、`cpu`（算力调度）
- 在 JS navItems 配置中添加对应路由

**验证方式**：页面加载后侧边栏显示 8 个导航项
