## 二、Git 自动化管控增强

### GIT-01: 提交前自动检查

**5大检查项**：
1. **Python语法检查** - compile()全量校验暂存区.py文件
2. **Import排序检查** - isort优先，降级简化模式
3. **大文件扫描** - 默认5MB阈值，超过阻止提交
4. **敏感信息扫描** - 10种正则模式（API Key、AWS Key、GitHub Token、私钥、JWT、数据库URL等），支持白名单注释
5. **关联单元测试** - 自动查找变更文件对应测试并运行

**提交信息检查**：
- Conventional Commits 规范
- 支持 type、scope、! 破坏性变更标记
- 类型：feat/fix/docs/style/refactor/perf/test/chore/release

**工具文件**：
- `scripts/git/run_precommit.py` - Python核心检查逻辑
- `scripts/git/pre-commit.ps1` - PowerShell钩子入口
- `scripts/git/commit-msg.ps1` - 提交信息校验
- `scripts/git/install_hooks.ps1` - 安装/卸载脚本

### GIT-05: M8管理台Git状态看板

**后端API（6个端点）**：

| 方法 | 路径 | 功能 | 权限 |
|------|------|------|------|
| GET | `/api/git/status` | 仓库状态概览 | 登录用户 |
| GET | `/api/git/commits` | 最近提交记录 | 登录用户 |
| GET | `/api/git/branches` | 分支列表 | 登录用户 |
| POST | `/api/git/commit` | 触发提交 | admin/owner |
| GET | `/api/git/diff` | 文件diff查看 | 登录用户 |
| GET | `/api/git/health` | Git健康检查 | 公开 |

**前端页面**：
- 4个KPI卡片（当前分支/变更文件数/本地分支数/最近提交）
- 最近提交时间线（20条，带作者、时间、hash）
- 变更文件列表（状态徽章，点击查看diff）
- 分支列表（高亮当前分支）
- 提交弹窗（格式校验、add_all选项）
- 30秒自动刷新 + 手动刷新按钮
- 深色主题 glass-surface 设计

## 四、手表交互基础接口

### 数据模型（3张表）
- **WatchDevice** - 手表设备表（设备ID/名称/类型/品牌/型号/固件/状态/电量/配对状态）
- **WatchHealthData** - 健康数据表（数据类型/数值/单位/时间戳/来源/质量）
- **WatchNotification** - 通知记录表（通知ID/标题/内容/类型/状态/送达时间/阅读时间）

### API接口（8个端点）

| 类别 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 设备管理 | GET | `/api/watch/devices` | 设备列表（分页/状态/类型过滤） |
| 设备管理 | POST | `/api/watch/devices` | 绑定设备 |
| 设备管理 | DELETE | `/api/watch/devices/{id}` | 解绑设备 |
| 健康数据 | GET | `/api/watch/health/realtime` | 实时健康数据（心率/步数/血氧/睡眠/卡路里） |
| 健康数据 | POST | `/api/watch/health/sync` | 同步健康数据 |
| 健康数据 | GET | `/api/watch/health/history` | 历史数据（分页/统计） |
| 通知推送 | POST | `/api/watch/notification/send` | 发送通知 |
| 通知推送 | GET | `/api/watch/notification/history` | 通知历史 |

### 实现特点
- 模拟数据驱动，预留真实SDK接入点
- 遵循现有路由代码风格
- ApiResponse 统一返回格式
- JWT鉴权保护
- 支持分页查询
