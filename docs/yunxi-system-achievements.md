# 云汐系统成果文档

**版本**：v1.1.0
**日期**：2026-07-08
**模块数**：10个（M1-M10）
**API端点**：600+
**前端页面**：55+

---

## 一、系统架构

### 1.1 模块总览

云汐系统采用微服务架构，共10个模块，通过 HTTP API 互相调用，由 M8 管理工作台统一纳管。

```
┌─────────────────────────────────────────────────────────┐
│                     M8 管理工作台                        │
│  (部署中心 / 监控中心 / Agent中心 / 算力调度 / 任务管理)  │
└────────────┬──────────────────────────┬─────────────────┘
             │                          │
    ┌────────┴────────┐        ┌────────┴────────┐
    │  M1 多Agent调度  │        │  M2 技能集群     │
    │  (联邦调度/8子Agent)│     │  (20+技能/语义检索)│
    └────────┬────────┘        └────────┬────────┘
             │                          │
    ┌────────┴────────┐        ┌────────┴────────┐
    │  M3 端云协同     │        │  M4 场景引擎     │
    │  (离线/同步/冲突)│        │  (模式切换/暖切换)│
    └────────┬────────┘        └────────┬────────┘
             │                          │
    ┌────────┴────────┐        ┌────────┴────────┐
    │  M5 潮汐记忆     │        │  M6 硬件外设     │
    │  (四层潮汐/加密) │        │  (手表/戒指/无人机)│
    └────────┬────────┘        └────────┬────────┘
             │                          │
    ┌────────┴────────┐        ┌────────┴────────┐
    │  M7 积木编排     │        │  M9 编程开发     │
    │  (可视化/DAG执行)│        │  (VSCode/代码沙箱)│
    └────────┬────────┘        └────────┬────────┘
             │                          │
    ┌────────┴──────────────────────────┴────────┐
    │          M10 系统卫士                       │
    │  (监控/防护/审计/沙箱调度/启动检查)         │
    └─────────────────────────────────────────────┘
```

### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | FastAPI + Python 3.10+ |
| 数据库 | SQLite + SQLAlchemy + Alembic |
| 前端 | 原生 HTML/CSS/JS + ECharts |
| 大模型 | DeepSeek / OpenAI / Ollama（多后端切换） |
| 认证 | JWT + API Key |
| 进程管理 | subprocess + 独立进程 |
| 配置管理 | pydantic-settings + .env + YAML |

## 二、各模块核心成果

### 2.1 M1 多Agent调度中心

**版本**：v11.1
**核心能力**：
- 联邦调度系统，支持 8 个子 Agent 协同工作
- DAG 任务编排，支持复杂任务分解和依赖管理
- 17 个联邦适配器，兼容多种大模型和 Agent 框架
- 成本控制器：Token 预算管理、成本追踪、优化建议
- 隐私卫士：数据脱敏、权限校验、审计追踪
- Agent 池管理：克隆工厂、实例池、负载均衡
- 生命周期管理：创建/运行/暂停/销毁全流程

**规模**：90+ Python文件，48个测试文件，30+架构文档

---

### 2.2 M2 技能集群

**版本**：v3.10.2
**核心能力**：
- 20+ 内置技能：日历、待办、翻译、代码搜索、数据分析、文档处理等
- BM25 语义检索 + 技能推荐引擎
- 技能流水线：多技能组合执行
- 沙箱执行：隔离的代码执行环境
- 插件系统：动态加载外部技能
- 缓存层：结果缓存 + 技能元数据缓存
- A2A 协议：Agent 间技能调用标准

**规模**：40+ Python文件，41个测试文件

---

### 2.3 M3 端云协同内核

**版本**：v2.1.2
**核心能力**：
- 端云双向同步：数据在本地和云端之间自动同步
- 离线缓存：断网时本地可用，联网后自动同步
- 冲突消解：基于时间戳 + 业务规则的冲突处理
- 云网关：统一的云端 API 入口
- 本地执行器：离线时本地执行计算
- 路由引擎：智能判断请求发往本地还是云端
- 显存监控：GPU 显存使用监控和调度

**规模**：30+ Python文件，18个测试文件

---

### 2.4 M4 业务场景引擎

**版本**：v2.9.0
**核心能力**：
- 六大业务模式：成长、生活、学习、工作、复盘、情绪
- 场景切换：模式间暖切换，过渡动画
- 上下文管理：跨场景状态保持
- 场景识别器：自动识别当前场景
- M8 标准对接：健康检查/指标/配置

**规模**：10个核心文件，3个测试文件

---

### 2.5 M5 潮汐记忆系统

**版本**：v2.4.0
**核心能力**：
- 四层潮汐记忆模型：L0沙滩（即时）/ L1浅水（短期）/ L2深水（长期）/ L3深海（永久）
- 睡眠巩固：自动将短期记忆转化为长期记忆
- 情绪引擎：情绪标签 + 情绪强度对记忆的影响
- 向量检索：语义相似度搜索
- AES-256 加密：记忆数据安全存储
- 三级域权限：个人/亲密/公共
- 审计日志：所有记忆操作留痕

**规模**：30+ Python文件，4个测试文件，5+技术文档

---

### 2.6 M6 穿戴硬件外设

**版本**：v7.0.0
**核心能力**：

- 传感器数据：心率、步数、睡眠、运动等模拟数据

- 设备控制：远程控制设备功能
- 通知服务：设备通知管理

**规模**：6类设备实现，20+ API端点

## 三、基础设施成果

### 3.1 shared/ 公共组件

- **全局配置管理**：YunxiConfig 单例，统一从 config/yunxi.env 加载
- **统一日志**：结构化 JSON 日志，多级别输出
- **LLM 客户端**：多后端切换（DeepSeek/OpenAI/Ollama），统一调用接口
- **模块间通信**：ModuleClient + ModuleRegistry，支持重试/超时/鉴权
- **进程管理**：ModuleProcessManager，支持所有模块启停管理

### 3.2 前端系统

- **55+ 页面**：覆盖管理台、模式页、积木平台、汐舷、启动页
- **暗色模式**：9个页面支持暗色主题切换
- **公共主题系统**：统一的 CSS 变量和主题切换
- **API 工具库**：统一的前端 API 调用封装
- **ECharts 可视化**：仪表盘数据可视化

### 3.3 数据库

- **61 张数据表**：覆盖用户、任务、模块、监控、成长、算力、进化、生活、复盘、学习、手表、工作、巡检等
- **SQLAlchemy ORM**：统一的数据模型定义
- **Alembic 迁移**：数据库版本管理（当前降级使用 create_all）
